1. Overview

Experiment tracking records configs, metrics, artifacts, models, and notes so you can compare runs and reproduce results. Without tracking, improvement becomes vibes-based engineering.

Mental model

An experiment run is a snapshot: code version, data version, hyperparameters, metrics, artifacts, and environment. If you cannot reproduce a result, you do not really own it.

What you should be able to do after this page

Track configurations, metrics, artifacts, model checkpoints, data versions, and compare experiments using MLflow or Weights & Biases style workflows.

3. Hands-on Code

MLflow basic tracking

Log parameters, metrics, and artifacts.

import mlflow

with mlflow.start_run():
    mlflow.log_param("learning_rate", 1e-3)
    mlflow.log_param("batch_size", 32)

    for epoch in range(5):
        train_loss = train_one_epoch()
        val_acc = evaluate()
        mlflow.log_metric("train_loss", train_loss, step=epoch)
        mlflow.log_metric("val_acc", val_acc, step=epoch)

    mlflow.log_artifact("best_model.pt")

Weights & Biases style tracking

W&B is popular for live dashboards and sweeps.

import wandb

run = wandb.init(project="image-classifier", config={"lr": 1e-3, "batch_size": 32})

for epoch in range(5):
    metrics = {"train_loss": train_one_epoch(), "val_acc": evaluate()}
    wandb.log(metrics, step=epoch)

wandb.save("best_model.pt")
run.finish()

Local JSON fallback

Even a simple log file is better than nothing.

import json, time

record = {
    "run_id": int(time.time()),
    "config": {"model": "mlp", "lr": 1e-3},
    "metrics": {"val_acc": 0.84},
    "artifact": "best_model.pt",
}
with open("run_log.jsonl", "a") as f:
    f.write(json.dumps(record) + "
")

4. Practical Workflow

Track only what helps decisions. A dashboard with 99 useless metrics is just expensive noise.

  1. Define baseline and primary metric.
  2. Log all hyperparameters that affect training.
  3. Log train and validation metrics per epoch.
  4. Save best model artifact and preprocessing metadata.
  5. Tag runs with dataset version and code commit.
  6. Compare runs using the same validation split.
  7. Promote only reproducible runs.

5. Common Mistakes

Traps to avoid
  • Changing data and model at the same time, then not knowing what helped.
  • Logging only final metrics without config.
  • Not recording random seed or dataset version.
  • Comparing runs that use different validation splits.
  • Keeping artifacts locally with no naming convention.
