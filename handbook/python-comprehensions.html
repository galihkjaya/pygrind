1. Mental Model: Build a Collection from an Iterable

A comprehension is a compact way to transform, filter, or construct a collection from another iterable. It is not just “shorter loop syntax”. It is a pattern for saying: “for every item here, produce something new”.

The core shape

new_list = [expression for item in iterable if condition]

# Read it like this:
# 1. for item in iterable
# 2. if condition is true
# 3. put expression into the new list
Use comprehensions when the goal is data construction If your loop mainly builds a new list, dict, or set, a comprehension may be clearer. If your loop has many side effects, logging, nested decisions, or error handling, a normal loop is usually better.

2. List Comprehensions

A list comprehension creates a new list. It is the most common type and often replaces a loop with append.

numbers = [1, 2, 3, 4, 5]

# Normal loop
squares = []
for n in numbers:
    squares.append(n * n)

# List comprehension
squares = [n * n for n in numbers]

print(squares)

Transform values

names = ["alya", "bima", "citra"]
capitalized = [name.title() for name in names]

prices = [10000, 25000, 50000]
prices_with_tax = [price * 1.11 for price in prices]

print(capitalized)
print(prices_with_tax)

Filter values

scores = [45, 80, 72, 90, 30]
passing_scores = [score for score in scores if score >= 70]

words = ["AI", "data", "ML", "python", "NLP"]
short_words = [word for word in words if len(word) <= 3]

print(passing_scores)
print(short_words)

3. Filtering vs Conditional Expressions

This part is a common trap. An if at the end filters items. An if/else before the for chooses what value to produce for every item.

Filter items

# Keep only passing scores
scores = [40, 75, 90]
passed = [s for s in scores if s >= 70]

Transform every item conditionally

# Label every score
scores = [40, 75, 90]
labels = ["pass" if s >= 70 else "fail" for s in scores]
⚠️ Different question, different syntax [x for x in items if condition] may return fewer items. [a if condition else b for x in items] returns the same number of items as the input.

4. Dict Comprehensions

A dict comprehension builds a dictionary. It is useful when you want to map keys to transformed values.

names = ["Alya", "Bima", "Citra"]
name_lengths = {name: len(name) for name in names}

print(name_lengths)

# Convert list of records into lookup table
students = [
    {"id": 101, "name": "Alya", "score": 88},
    {"id": 102, "name": "Bima", "score": 76},
]

students_by_id = {student["id"]: student for student in students}
print(students_by_id[101])

Filtering in dict comprehensions

scores = {
    "Alya": 88,
    "Bima": 61,
    "Citra": 92,
    "Deni": 55,
}

passed = {name: score for name, score in scores.items() if score >= 70}
print(passed)
❌ Duplicate keys overwrite earlier values If two items produce the same key, the later value wins. This is normal dict behavior, but can silently remove data. 
words = ["hi", "to", "cat"]
by_length = {len(word): word for word in words}
print(by_length)  # length 2 keeps only the last length-2 word

5. Set Comprehensions

A set comprehension builds unique values. It is useful for deduplication and extracting distinct features from data.

emails = [
    "ALYA@example.com",
    "bima@example.com",
    "alya@example.com",
]

normalized_emails = {email.lower() for email in emails}
print(normalized_emails)

transactions = [
    {"category": "food", "amount": 25000},
    {"category": "transport", "amount": 12000},
    {"category": "food", "amount": 18000},
]

categories = {trx["category"] for trx in transactions}
print(categories)

6. Generator Expressions: Lazy Comprehensions

A generator expression looks like a list comprehension with parentheses, but it does not build the whole list immediately. It produces values one by one. This is useful for large data or when feeding values into functions like sum, any, and all.

numbers = [1, 2, 3, 4, 5]

# Builds a full list first
squares_list = [n * n for n in numbers]

# Produces values lazily
squares_gen = (n * n for n in numbers)

print(sum(n * n for n in numbers))
print(any(n > 3 for n in numbers))
print(all(n > 0 for n in numbers))
List comprehension vs generator expression Use a list comprehension when you need the resulting list multiple times, need indexing, or need to inspect it. Use a generator expression when you only stream values once into another operation.

7. Nested Comprehensions

Nested comprehensions are powerful but easy to read wrong. The order of for clauses follows the same order as nested loops.

matrix = [
    [1, 2, 3],
    [4, 5, 6],
]

# Normal nested loop
flattened = []
for row in matrix:
    for value in row:
        flattened.append(value)

# Equivalent nested comprehension
flattened = [value for row in matrix for value in row]

print(flattened)

Matrix transformation

matrix = [
    [1, 2, 3],
    [4, 5, 6],
]

# Square every cell but keep the 2D shape
squared_matrix = [[value * value for value in row] for row in matrix]

print(squared_matrix)
⚠️ Readability rule One loop: usually good. Two loops: okay if simple. Three loops or complex logic: use normal loops. Your future self deserves human rights.

8. Practical Patterns

Pattern A: Clean text data

raw_tags = [" Python ", "DATA", "", " ml ", "Python"]

clean_tags = [tag.strip().lower() for tag in raw_tags if tag.strip()]
unique_tags = {tag.strip().lower() for tag in raw_tags if tag.strip()}

print(clean_tags)
print(unique_tags)

Pattern B: Extract columns from rows

rows = [
    {"name": "Alya", "score": 88},
    {"name": "Bima", "score": 76},
    {"name": "Citra", "score": 92},
]

names = [row["name"] for row in rows]
scores = [row["score"] for row in rows]
score_by_name = {row["name"]: row["score"] for row in rows}

print(names)
print(scores)
print(score_by_name)

Pattern C: Combine two lists

names = ["Alya", "Bima", "Citra"]
scores = [88, 76, 92]

records = [{"name": name, "score": score} for name, score in zip(names, scores)]
print(records)

9. Readability and Style

A comprehension is good when it makes the data transformation obvious. It is bad when it becomes a puzzle. Compact does not automatically mean clean. Kadang kode pendek itu bukan elegan, cuma typo yang lebih susah dilacak.

SituationPreferWhy
Simple transformComprehensionClear and compact
Simple filterComprehensionReadable condition
Multiple side effectsNormal loopEasier to debug
Try/except inside loopNormal loopComprehensions do not handle multi-line exception logic well
Deep nestingNormal loopLess mental overhead

10. Common Mistakes

❌ Using comprehension only for side effects 
# Bad: creates a useless list of None values
[print(name) for name in names]

# Good
for name in names:
    print(name)
❌ Mixing up filter syntax and conditional expression syntax 
scores = [40, 80, 90]

# Filter: fewer items
passed_scores = [s for s in scores if s >= 70]

# Conditional expression: same number of items
labels = ["pass" if s >= 70 else "fail" for s in scores]
⚠️ Overusing nested comprehensions If you need to explain the comprehension line for more than 10 seconds, it probably wants to become a normal loop.
