1. Overview

Connect Python apps safely using psycopg, asyncpg, pooling, and parameterized queries. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

4. Hands-on Code

Parameterized query

Never build SQL values using f-strings.

import psycopg2

conn = psycopg2.connect('dbname=app user=postgres password=secret')
with conn:
    with conn.cursor() as cur:
        cur.execute(
            'SELECT id, email FROM users WHERE country = %s LIMIT %s',
            ('Indonesia', 10),
        )
        print(cur.fetchall())

Manual transaction

Rollback if any step fails.

try:
    cur.execute('UPDATE accounts SET balance = balance - %s WHERE id = %s', (50000, 1))
    cur.execute('UPDATE accounts SET balance = balance + %s WHERE id = %s', (50000, 2))
    conn.commit()
except Exception:
    conn.rollback()
    raise

asyncpg pool

Async web apps should reuse connections.

pool = await asyncpg.create_pool(dsn=DATABASE_URL)

async with pool.acquire() as conn:
    rows = await conn.fetch(
        'SELECT id, email FROM users WHERE country = $1 LIMIT $2',
        'Indonesia', 10,
    )

5. Practical Workflow

  1. Store credentials in environment variables.
  2. Use a pool in web apps.
  3. Use parameterized queries for values.
  4. Wrap multi-step writes in transactions.
  5. Close resources with context managers.
  6. Log errors without leaking secrets.

6. Common Mistakes

Avoid these traps
  • SQL injection through f-strings.
  • Opening a new connection per request.
  • Forgetting rollback after failed writes.
  • Hardcoding database passwords.
  • Mixing sync drivers into async routes under load.
