1. Overview

Hugging Face provides models, tokenizers, datasets, and training utilities for transformer workflows. It makes pretrained models accessible while still requiring careful data handling and evaluation.

Mental model

A Hugging Face workflow usually has four objects: dataset, tokenizer, model, and trainer or pipeline. The tokenizer must match the model. The model head must match the task.

What you should be able to do after this page

Use pipelines for quick inference, tokenize datasets, fine-tune with Trainer, evaluate predictions, and understand the difference between checkpoint, tokenizer, config, and model head.

3. Hands-on Code

Quick pipeline inference

Great for prototypes. Not enough for serious evaluation.

from transformers import pipeline

clf = pipeline("sentiment-analysis")
print(clf("I love this course, it finally makes sense."))

Tokenize a batch

Tokenizer output is what the model actually consumes.

from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased")
texts = ["Transformers are useful", "Tokenization matters"]
encoded = tokenizer(texts, padding=True, truncation=True, return_tensors="pt")
print(encoded.keys())
print(encoded["input_ids"].shape)

Fine-tuning skeleton

Trainer reduces boilerplate but does not remove responsibility for good validation.

from transformers import AutoModelForSequenceClassification, TrainingArguments, Trainer

model = AutoModelForSequenceClassification.from_pretrained("distilbert-base-uncased", num_labels=2)
args = TrainingArguments(
    output_dir="runs/text-clf",
    eval_strategy="epoch",
    save_strategy="epoch",
    learning_rate=2e-5,
    per_device_train_batch_size=16,
    num_train_epochs=3,
)
trainer = Trainer(model=model, args=args, train_dataset=train_ds, eval_dataset=valid_ds, tokenizer=tokenizer)
trainer.train()

4. Practical Workflow

Treat Hugging Face as a clean workflow, not a magic button.

  1. Pick checkpoint appropriate to language and task.
  2. Load matching tokenizer and model class.
  3. Create train/validation split.
  4. Tokenize with padding and truncation.
  5. Set labels and metrics.
  6. Train with Trainer or custom loop.
  7. Save model and tokenizer together.

5. Common Mistakes

Traps to avoid
  • Using tokenizer from one checkpoint with a different model checkpoint.
  • Forgetting truncation and hitting sequence length errors.
  • Using pipeline outputs as evaluation without a test set.
  • Ignoring label mapping order.
  • Saving model without tokenizer and config.
