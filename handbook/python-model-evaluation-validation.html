Model Evaluation & Validation

Know whether your model is actually good or just vibing with leakage.

Supports ML Core
OverviewConceptsCodeWorkflowMistakesQuick refPractice

1. Overview

Evaluation tells you whether a model generalizes beyond training data. Validation design is often more important than model choice because a bad split can make weak models look impressive.

Mental model

Evaluation is a simulation of future use. Your validation data should resemble the data the model will see after deployment. If the split is unrealistic, the metric is basically fanfiction.

What you should be able to do after this page

Understand train/validation/test splits, cross-validation, overfitting, classification metrics, regression metrics, threshold tuning, and leakage checks.

3. Hands-on Code

Classification metrics

Accuracy can hide failure under class imbalance.

from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score, roc_auc_score

proba = model.predict_proba(X_test)[:, 1]
pred = proba >= 0.5
print("accuracy", accuracy_score(y_test, pred))
print("precision", precision_score(y_test, pred))
print("recall", recall_score(y_test, pred))
print("f1", f1_score(y_test, pred))
print("roc_auc", roc_auc_score(y_test, proba))

Regression metrics

MAE is easy to interpret. RMSE punishes large errors. R2 compares to a baseline.

from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score

pred = model.predict(X_test)
rmse = mean_squared_error(y_test, pred, squared=False)
print(mean_absolute_error(y_test, pred), rmse, r2_score(y_test, pred))

Cross-validation

Use stratified CV for classification when class balance matters.

from sklearn.model_selection import StratifiedKFold, cross_val_score

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(pipe, X, y, cv=cv, scoring="f1")
print(scores.mean(), scores.std())

Threshold tuning

Tune threshold on validation data, not test data.

import numpy as np
from sklearn.metrics import f1_score

for t in np.linspace(0.1, 0.9, 17):
    pred = val_proba >= t
    print(t, f1_score(y_val, pred))

4. Practical Workflow

Validation must mirror the real problem. Time series, grouped users, repeated patients, and duplicated content require special split strategies.

  1. Choose metrics based on error cost and target type.
  2. Create a split strategy matching deployment reality.
  3. Use validation for model selection and threshold tuning.
  4. Use test set only once for final reporting.
  5. Compare with a simple baseline.
  6. Report uncertainty such as CV mean and standard deviation.
  7. Perform error analysis.

5. Common Mistakes

Traps to avoid
  • Tuning on the test set. That is cheating with extra steps.
  • Using random split when time order matters.
  • Using accuracy alone for imbalanced classification.
  • Forgetting grouped leakage, such as same user in train and test.
  • Reporting metrics without baseline comparison.
