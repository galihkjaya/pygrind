1. Overview

You do not need to become a pure mathematician to build ML systems, but you do need core intuitions. Statistics reasons from samples, probability models uncertainty, linear algebra represents high-dimensional data, and calculus explains optimization.

Mental model

Vectors describe observations, matrices describe many observations, probability describes uncertainty, statistics estimates patterns from samples, and calculus tells models how to adjust themselves.

What you should be able to do after this page

Understand mean, variance, distributions, conditional probability, vectors, matrices, dot products, gradients, and why optimization is central to machine learning.

3. Hands-on Code

Descriptive statistics

Mean gives center. Standard deviation gives spread. Both are summaries, not full reality.

import numpy as np

x = np.array([10, 12, 13, 15, 100])
print("mean:", x.mean())
print("median:", np.median(x))
print("std:", x.std())

Probability by simulation

Simulation helps build probability intuition before formulas.

import random

def coin_flip():
    return random.choice(["H", "T"])

flips = [coin_flip() for _ in range(1000)]
print(flips.count("H") / len(flips))

Vectors and dot products

A dot product can act like weighted scoring or similarity depending on context.

import numpy as np

x = np.array([2.0, 4.0, 1.0])
w = np.array([0.5, -0.2, 1.5])
score = x @ w
print(score)

Gradient descent intuition

The learning rate controls how far parameters move against the gradient.

w = 0.0
learning_rate = 0.1

for step in range(10):
    gradient = 2 * (w - 3)    # derivative of (w - 3)^2
    w = w - learning_rate * gradient
    print(step, w)

4. Practical Workflow

Use math as a diagnostic tool. When a model behaves strangely, ask whether the issue is distribution, probability assumption, vector representation, or optimization dynamics.

  1. Summarize variables with center and spread.
  2. Inspect distributions and outliers before assuming normality.
  3. Use probability to reason about uncertainty.
  4. Represent rows as vectors and datasets as matrices.
  5. Define a loss function for “bad prediction”.
  6. Use gradients or search methods to reduce loss.

5. Common Mistakes

Traps to avoid
  • Memorizing formulas without knowing what they measure.
  • Treating correlation as causation.
  • Ignoring feature scale in distance-based models.
  • Thinking high accuracy always means a good model under imbalance.
  • Using calculus mechanically without understanding loss.
