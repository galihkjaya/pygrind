Supervised Learning: Regression, Classification, Trees & Boosting

Predict numeric values or labels from examples with known answers.

1. Overview

Supervised learning trains from labeled examples. Regression predicts continuous values, classification predicts categories, trees learn rule-based splits, and boosting combines many weak learners into strong models.

Mental model

A supervised model learns a mapping from features X to target y. Linear models prefer additive relationships, trees split feature space, and boosting builds corrections step by step.

What you should be able to do after this page

Understand common supervised model families, when to use them, how to train them in scikit-learn, and how to interpret strengths and limits.

3. Hands-on Code

Regression baseline

Always compare with a simple baseline.

from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_absolute_error, r2_score

reg = LinearRegression()
reg.fit(X_train, y_train)
pred = reg.predict(X_test)
print(mean_absolute_error(y_test, pred))
print(r2_score(y_test, pred))

Classification baseline

For imbalanced data, inspect precision, recall, F1, and probabilities.

from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report, roc_auc_score

clf = LogisticRegression(max_iter=1000, class_weight="balanced")
clf.fit(X_train, y_train)
proba = clf.predict_proba(X_test)[:, 1]
pred = clf.predict(X_test)
print(classification_report(y_test, pred))
print(roc_auc_score(y_test, proba))

Random forest

Trees are interpretable-ish but can overfit. Forests average many trees.

from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(n_estimators=300, min_samples_leaf=5, random_state=42, n_jobs=-1)
rf.fit(X_train, y_train)
print(rf.score(X_test, y_test))

Gradient boosting

Often strong on tabular data. Tune carefully and validate honestly.

from sklearn.ensemble import HistGradientBoostingClassifier

gb = HistGradientBoostingClassifier(learning_rate=0.05, max_iter=300, random_state=42)
gb.fit(X_train, y_train)
print(gb.score(X_test, y_test))

4. Practical Workflow

Start with simple models before complex models. If a baseline beats your fancy monster, the monster is just expensive decoration.

  1. Frame the task: regression or classification.
  2. Choose metric based on error cost.
  3. Build a simple baseline first.
  4. Train several model families under the same split.
  5. Inspect coefficients or feature importance cautiously.
  6. Perform error analysis.
  7. Tune only promising models.

5. Common Mistakes

Traps to avoid
  • Using regression metrics for classification or vice versa.
  • Trusting feature importance as causal explanation.
  • Letting deep trees overfit tiny datasets.
  • Forgetting to stratify classification splits.
  • Reporting only the best model without baseline comparison.
