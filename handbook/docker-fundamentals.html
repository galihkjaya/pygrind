1. Overview

Understand images, containers, registries, logs, exec, and lifecycle commands. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

2. Mental Model

How to think about this topic

The core mental model: every command or construct has an input, a transformation, and an output. Strong engineers do not just copy syntax. They predict what should happen, run the smallest possible check, then scale the pattern carefully.

Learning target

After this page, you should be able to explain the purpose of the topic, write the main syntax from scratch, debug common failure modes, and apply it inside a small backend or AI-engineering project.

4. Hands-on Code

Run a container

Run an image and map a port.

docker run --name web -p 8080:80 nginx

Inspect containers

Know what is running before debugging.

docker ps
docker ps -a
docker logs web
docker stop web
docker rm web

Exec into container

Use exec for quick inspection inside a running container.

docker exec -it web sh
ls
exit

5. Practical Workflow

  1. Learn image vs container first.
  2. Run simple images before custom builds.
  3. Name containers for easier commands.
  4. Read logs before guessing.
  5. Stop and remove unused containers.
  6. Use docker system prune carefully.

6. Common Mistakes

Avoid these traps
  • Confusing image and container.
  • Forgetting port mapping.
  • Debugging without reading logs.
  • Leaving many stopped containers around.
  • Running prune without understanding what will be deleted.
