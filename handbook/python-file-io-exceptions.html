File I/O & Exception Handling

Read, write, validate, and protect your Python programs from messy real-world files and runtime errors.

1. Mental Model: Programs Talk to the Outside World

Variables disappear when a program ends. Files let your program store data permanently, read existing data, exchange data with other tools, and recover from errors. Exception handling lets your program fail gracefully instead of exploding like a group project one hour before deadline.

Two separate skills

  • File I/O: opening, reading, writing, appending, and closing files.
  • Exception handling: deciding what should happen when something goes wrong.

2. Paths and Files

A file path tells Python where the file lives. You can use relative paths from your current working directory or absolute paths from the root of your system.

from pathlib import Path

# Current working directory
print(Path.cwd())

# Relative path
notes_path = Path("notes.txt")

# Folder + file path
project_dir = Path("data")
users_path = project_dir / "users.csv"

print(users_path)
Use pathlib for modern Python pathlib.Path makes path handling cleaner across Windows, macOS, and Linux. It also avoids messy manual string joining.

3. Opening and Reading Files

The safest beginner pattern is using with open(...). It automatically closes the file after the block, even when an error happens.

# Read the entire file as one string
with open("notes.txt", "r", encoding="utf-8") as file:
    content = file.read()

print(content)

Read line by line

# Good for large files because it does not load everything at once
with open("notes.txt", "r", encoding="utf-8") as file:
    for line in file:
        clean_line = line.strip()
        print(clean_line)

Read all lines into a list

with open("notes.txt", "r", encoding="utf-8") as file:
    lines = file.readlines()

print(lines)

4. Writing and Appending Files

Writing modes determine whether Python creates, overwrites, or appends to a file. This is where many beginners accidentally Thanos-snap their data.

# "w" creates a new file or overwrites an existing file
with open("output.txt", "w", encoding="utf-8") as file:
    file.write("Hello, Python!
")
    file.write("This is a new file.
")

# "a" appends to the end of a file
with open("output.txt", "a", encoding="utf-8") as file:
    file.write("This line is appended.
")

Create folders before writing

from pathlib import Path

output_dir = Path("reports")
output_dir.mkdir(exist_ok=True)

report_path = output_dir / "summary.txt"

with open(report_path, "w", encoding="utf-8") as file:
    file.write("Training summary goes here.
")

"rb"/"wb"Binary read/writeFor images, PDFs, models, etc.

5. CSV and JSON: Files You Will Actually Use

Plain text files are useful, but CSV and JSON are the daily drivers for data and APIs.

CSV: rows and columns

import csv

rows = [
    {"name": "Alya", "score": 88},
    {"name": "Bima", "score": 76},
]

with open("scores.csv", "w", newline="", encoding="utf-8") as file:
    writer = csv.DictWriter(file, fieldnames=["name", "score"])
    writer.writeheader()
    writer.writerows(rows)

with open("scores.csv", "r", encoding="utf-8") as file:
    reader = csv.DictReader(file)
    for row in reader:
        print(row["name"], row["score"])

JSON: nested data

import json

config = {
    "model": "logistic_regression",
    "threshold": 0.5,
    "features": ["age", "income", "score"],
}

with open("config.json", "w", encoding="utf-8") as file:
    json.dump(config, file, indent=2)

with open("config.json", "r", encoding="utf-8") as file:
    loaded_config = json.load(file)

print(loaded_config["features"])
⚠️ CSV values are read as strings When you read from CSV, numbers often arrive as text. Convert them manually before doing math.

6. Exceptions: Handling Errors Gracefully

An exception is Python saying: “I cannot continue normally.” Without handling, your program stops. With handling, you can show a useful message, use a fallback, retry, or clean up resources.

try:
    number = int(input("Enter a number: "))
    result = 100 / number
except ValueError:
    print("Please enter a valid integer.")
except ZeroDivisionError:
    print("Cannot divide by zero.")
else:
    print("Result:", result)
finally:
    print("Done.")
BlockRuns when?Purpose
tryAlways attempted firstPut risky code here
exceptOnly if matching error happensRecover or explain
elseOnly if no exception happenedCode that depends on success

7. Safe File Patterns

Pattern A: Read file if it exists

from pathlib import Path

path = Path("notes.txt")

if path.exists():
    content = path.read_text(encoding="utf-8")
    print(content)
else:
    print("notes.txt does not exist yet.")

Pattern B: Try reading JSON with useful errors

import json
from pathlib import Path

path = Path("config.json")

try:
    with open(path, "r", encoding="utf-8") as file:
        config = json.load(file)
except FileNotFoundError:
    print("Config file is missing.")
    config = {}
except json.JSONDecodeError:
    print("Config file is not valid JSON.")
    config = {}

print(config)

Pattern C: Validate data and raise your own error

def load_threshold(config):
    threshold = config.get("threshold")

    if threshold is None:
        raise ValueError("Missing threshold in config")

    if not 0 <= threshold <= 1:
        raise ValueError("Threshold must be between 0 and 1")

    return threshold
Raise exceptions when bad data should not continue Printing an error is not enough if the program would continue with wrong assumptions. Raise an exception when continuing would produce misleading results.

8. Custom Exceptions

For larger projects, custom exceptions make error meaning clearer. They help separate domain errors from generic Python errors.

class InvalidScoreError(Exception):
    pass


def normalize_score(score):
    if not 0 <= score <= 100:
        raise InvalidScoreError("Score must be between 0 and 100")
    return score / 100

try:
    print(normalize_score(120))
except InvalidScoreError as error:
    print("Invalid score:", error)

9. Debugging File and Exception Problems

  • Print Path.cwd() when Python “cannot find” your file.
  • Use encoding="utf-8" consistently for text files.
  • Use newline="" when writing CSV files.
  • Catch specific exceptions first. Broad except Exception should be rare.
  • Do not hide errors silently. At least log or print a meaningful message.

10. Common Mistakes

❌ Forgetting that "w" overwrites files 
# This destroys existing content if the file already exists
with open("important.txt", "w", encoding="utf-8") as file:
    file.write("new content")

# Use "a" to append, or "x" to fail if file exists
❌ Catching every exception and doing nothing 
try:
    risky_operation()
except Exception:
    pass  # Bad: hides the problem

# Better
try:
    risky_operation()
except ValueError as error:
    print("Invalid value:", error)
⚠️ Opening files without with

You can manually close files, but with is safer and cleaner. Use it as your default.
