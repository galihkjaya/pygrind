1. Overview

Map URLs and HTTP methods to Python functions cleanly. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

2. Mental Model

How to think about this topic

The core mental model: every command or construct has an input, a transformation, and an output. Strong engineers do not just copy syntax. They predict what should happen, run the smallest possible check, then scale the pattern carefully.

Learning target

After this page, you should be able to explain the purpose of the topic, write the main syntax from scratch, debug common failure modes, and apply it inside a small backend or AI-engineering project.

4. Hands-on Code

Basic app

A route is just a Python function exposed over HTTP.

from fastapi import FastAPI

app = FastAPI(title='Learning API')

@app.get('/')
def root():
    return {'message': 'Hello API'}

@app.get('/users/{user_id}')
def get_user(user_id: int):
    return {'user_id': user_id}

Query parameters

Use query params for filtering, search, pagination, and optional modifiers.

@app.get('/courses')
def list_courses(q: str | None = None, limit: int = 20, offset: int = 0):
    return {'q': q, 'limit': limit, 'offset': offset}

Method semantics

Keep behavior aligned with method names.

@app.post('/courses', status_code=201)
def create_course():
    return {'status': 'created'}

@app.delete('/courses/{course_id}')
def delete_course(course_id: int):
    return {'deleted': course_id}

5. Practical Workflow

  1. Name resources as nouns.
  2. Use GET for read, POST for create, PATCH for partial update, DELETE for delete.
  3. Use path params for identity.
  4. Use query params for filtering and paging.
  5. Type parameters for validation.
  6. Test routes in /docs.

6. Common Mistakes

Avoid these traps
  • Using verbs like /getUser instead of resource routes.
  • Putting filters inside path params.
  • Returning inconsistent response shapes.
  • Forgetting status_code for creation.
  • Making one endpoint do everything.
