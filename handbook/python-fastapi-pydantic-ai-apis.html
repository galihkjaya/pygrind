FastAPI & Pydantic for AI APIs

Wrap AI features in clean, typed HTTP APIs.

1. Overview

FastAPI is a Python web framework for building APIs quickly, and Pydantic validates request and response data. Together they are excellent for AI services that need typed inputs, structured outputs, and clear error handling.

Mental model

An AI API is a contract. The client sends validated input, the server runs retrieval/model/business logic, and the response returns a predictable schema. Pydantic keeps the boundary clean.

What you should be able to do after this page

Build FastAPI endpoints, define Pydantic models, validate inputs, return structured responses, handle errors, and organize AI service code.

3. Hands-on Code

Minimal FastAPI app

Typed input and output make APIs safer.

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field

app = FastAPI(title="AI Tutor API")

class ReviewRequest(BaseModel):
    code: str = Field(min_length=1)
    problem: str

class ReviewResponse(BaseModel):
    passed: bool
    feedback: str
    score: int

@app.post("/review", response_model=ReviewResponse)
def review_code(req: ReviewRequest):
    if "import os" in req.code:
        raise HTTPException(status_code=400, detail="Unsafe code pattern")
    return ReviewResponse(passed=True, feedback="Looks good structurally.", score=85)

Service layer pattern

Keep model/provider logic outside route functions.

class TutorService:
    def __init__(self, llm_client):
        self.llm_client = llm_client

    def review(self, code: str, problem: str) -> dict:
        prompt = f"Problem: {problem}
Code:
{code}"
        return self.llm_client.generate_json(prompt)

service = TutorService(llm_client)

4. Practical Workflow

Production API code should separate schemas, routes, services, providers, and configuration.

  1. Define request and response schemas first.
  2. Implement pure service logic separate from FastAPI routes.
  3. Validate inputs with Pydantic constraints.
  4. Handle provider errors with clear HTTP responses.
  5. Add logging and request IDs.
  6. Write tests for endpoints.
  7. Document environment variables and deployment command.

5. Common Mistakes

Traps to avoid
  • Returning unstructured dictionaries everywhere.
  • Putting all LLM logic inside route functions.
  • Leaking API keys in logs or responses.
  • No timeout around model/provider calls.
  • No tests for error cases.
