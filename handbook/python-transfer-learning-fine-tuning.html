Transfer Learning & Fine-tuning

Start from a pretrained model instead of training from zero like a medieval monk.

Supports Deep Learning Core

1. Overview

Transfer learning reuses knowledge from a model trained on a large dataset. Fine-tuning adapts that model to your task. This is often cheaper, faster, and more accurate than training from scratch.

Mental model

A pretrained model is a feature extractor plus a task-specific head. You can freeze the backbone, train a new head, then optionally unfreeze some layers for careful fine-tuning.

What you should be able to do after this page

Understand freezing, feature extraction, classifier heads, learning rates, gradual unfreezing, checkpoints, and overfitting risks with small datasets.

3. Hands-on Code

Freeze a backbone

Train only the final head first.

import torch
from torch import nn
from torchvision import models

model = models.resnet18(weights="DEFAULT")
for p in model.parameters():
    p.requires_grad = False

num_features = model.fc.in_features
model.fc = nn.Linear(num_features, 3)
optimizer = torch.optim.Adam(model.fc.parameters(), lr=1e-3)

Unfreeze selected layers

Fine-tune carefully with a smaller learning rate.

for name, p in model.named_parameters():
    if name.startswith("layer4") or name.startswith("fc"):
        p.requires_grad = True
    else:
        p.requires_grad = False

optimizer = torch.optim.Adam(filter(lambda p: p.requires_grad, model.parameters()), lr=1e-5)

Checkpoint best validation model

Save the best model, not just the last model.

best_val = 0.0
for epoch in range(num_epochs):
    train_one_epoch(model, train_loader, optimizer)
    val_score = evaluate(model, val_loader)
    if val_score > best_val:
        best_val = val_score
        torch.save(model.state_dict(), "best_model.pt")

4. Practical Workflow

Fine-tuning works best when you control overfitting. Small datasets need stronger validation, augmentation, and conservative learning rates.

  1. Pick a pretrained backbone close to your data type.
  2. Replace the task head.
  3. Freeze backbone and train the head.
  4. Evaluate and inspect errors.
  5. Unfreeze later layers if needed.
  6. Use lower learning rate for pretrained layers.
  7. Save best checkpoint and record preprocessing.

5. Common Mistakes

Traps to avoid
  • Using preprocessing different from what the pretrained model expects.
  • Fine-tuning all layers immediately on a tiny dataset.
  • Using too high a learning rate and destroying pretrained features.
  • Saving the last checkpoint instead of best validation checkpoint.
  • Ignoring domain shift and label quality.
