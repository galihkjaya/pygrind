RAG: Embeddings, Vector Databases & Retrieval Pipelines

Give models external knowledge instead of hoping they remember everything.

1. Overview

Retrieval-Augmented Generation connects an LLM to external documents. Instead of relying only on model memory, the app retrieves relevant context and asks the model to answer from that context.

Mental model

RAG has two stages: indexing and answering. Indexing chunks documents, embeds them, and stores vectors. Answering embeds the query, retrieves relevant chunks, builds a prompt, and generates a grounded response.

What you should be able to do after this page

Understand chunking, embeddings, vector search, metadata, retrieval, reranking, context construction, citations, and RAG evaluation.

3. Hands-on Code

Index documents

Indexing happens offline or whenever documents change.

docs = load_documents("handbook/")
chunks = []
for doc in docs:
    for chunk in split_into_chunks(doc.text, max_tokens=500):
        chunks.append({"text": chunk, "source": doc.path})

embeddings = embed([c["text"] for c in chunks])
vector_store.add(embeddings, metadata=chunks)

Retrieve and answer

At query time, retrieve before generating.

query = "How do Python generators work?"
q_emb = embed([query])
results = vector_store.search(q_emb, top_k=5)

context = "

".join(r["text"] for r in results)
prompt = f"""
Answer using only the context. If missing, say you do not know.

Context:
{context}

Question: {query}
"""
answer = llm(prompt)
print(answer)

Citation-friendly context

Keep source metadata attached to every chunk.

for i, r in enumerate(results, start=1):
    print(f"[{i}] source={r['source']} score={r['score']}")
    print(r["text"][:300])

4. Practical Workflow

RAG quality is mostly retrieval quality. The generator cannot use evidence it never receives.

  1. Collect documents and preserve metadata.
  2. Chunk by semantic boundaries, not random cuts.
  3. Embed chunks and store vectors.
  4. Retrieve top-k chunks for the query.
  5. Optionally rerank results.
  6. Build prompt with context and citation instructions.
  7. Evaluate retrieval and answer faithfulness separately.

5. Common Mistakes

Traps to avoid
  • Chunks too large, causing vague retrieval.
  • Chunks too small, losing context.
  • No metadata, so citations become impossible.
  • Assuming vector similarity means factual correctness.
  • Evaluating only final answer and ignoring retrieval failures.
