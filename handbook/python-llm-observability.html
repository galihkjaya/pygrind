LLM Observability: Traces, Logs, LangSmith & Langfuse

Debug AI behavior with evidence, not vibes and screenshots.

1. Overview

LLM observability records prompts, retrieved context, model outputs, tool calls, latency, errors, and cost. It helps debug failures, monitor regressions, and evaluate production quality.

Mental model

A trace is the story of one request: user input, retrieval, tool calls, model calls, validations, and final response. Without traces, multi-step AI systems become haunted houses.

What you should be able to do after this page

Understand traces, spans, metadata, prompt/version logging, retrieval logging, cost tracking, feedback, evaluation datasets, and privacy concerns.

3. Hands-on Code

Manual trace object

Even simple structured logs are useful.

import time, uuid, json

trace = {
    "trace_id": str(uuid.uuid4()),
    "started_at": time.time(),
    "steps": [],
}

trace["steps"].append({"name": "retrieve", "top_k": 5, "latency_ms": 42})
trace["steps"].append({"name": "llm", "model": "example-model", "input_tokens": 800, "output_tokens": 120})

print(json.dumps(trace, indent=2))

Decorate spans

Wrap expensive steps to record latency and errors.

import time

def traced_step(name):
    def deco(fn):
        def wrapper(*args, **kwargs):
            start = time.time()
            try:
                result = fn(*args, **kwargs)
                status = "ok"
                return result
            except Exception:
                status = "error"
                raise
            finally:
                print({"step": name, "status": status, "latency_ms": (time.time()-start)*1000})
        return wrapper
    return deco

4. Practical Workflow

Log enough to debug but not so much that you create privacy or cost problems.

  1. Assign a trace_id to each request.
  2. Record model, prompt version, latency, token usage, and errors.
  3. Record retrieved chunks and source metadata for RAG.
  4. Capture tool calls and validation failures.
  5. Collect user feedback and evaluator labels.
  6. Redact secrets and sensitive data.
  7. Build dashboards for quality, cost, and reliability.

5. Common Mistakes

Traps to avoid
  • Logging raw secrets or private user data.
  • No prompt versioning, making regressions mysterious.
  • Tracking cost but not quality.
  • Tracking final answer but not retrieved context.
  • No way to connect user feedback to traces.
