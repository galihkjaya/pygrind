CNNs, RNNs & LSTMs

Understand model families for images and sequences.

Supports Deep Learning Core

1. Overview

CNNs specialize in spatial patterns such as images. RNNs and LSTMs specialize in ordered sequences such as text, audio, or time series. Each architecture encodes assumptions about structure.

Mental model

CNNs slide filters over local neighborhoods. RNNs process sequences step by step while carrying hidden state. LSTMs add gates to remember or forget information more reliably across longer sequences.

What you should be able to do after this page

Understand convolutions, pooling, channels, recurrent hidden state, LSTM gates at a high level, and when each architecture family is appropriate.

3. Hands-on Code

Simple CNN block

A CNN reduces images into learned feature maps.

import torch
from torch import nn

cnn = nn.Sequential(
    nn.Conv2d(in_channels=3, out_channels=16, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.MaxPool2d(2),
    nn.Conv2d(16, 32, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.AdaptiveAvgPool2d((1, 1)),
    nn.Flatten(),
    nn.Linear(32, 10),
)

x = torch.randn(8, 3, 64, 64)
print(cnn(x).shape)

LSTM classifier skeleton

For sequence tasks, the final hidden state can summarize the sequence.

class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_dim, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, num_classes)

    def forward(self, tokens):
        x = self.embedding(tokens)
        output, (h_n, c_n) = self.lstm(x)
        final_hidden = h_n[-1]
        return self.fc(final_hidden)

Shape check habit

Most architecture bugs are shape bugs wearing a hoodie.

images = torch.randn(4, 3, 128, 128)

print("image batch:", images.shape)
print("token batch:", tokens.shape)

4. Practical Workflow

Choose architecture based on data structure: spatial grids, ordered sequences, or general tabular vectors.

  1. Identify data shape and task.
  2. For images, use CNN blocks or pretrained vision backbones.
  3. For sequences, decide whether recurrent models or transformers fit better.
  4. Track shapes after each major layer.
  5. Use batching and padding for variable-length sequences.
  6. Evaluate with task-specific metrics.

5. Common Mistakes

Traps to avoid
  • Feeding images with wrong dimension order. PyTorch expects NCHW.
  • Flattening images too early and destroying spatial structure.
  • Ignoring padding/masks for variable-length sequences.
  • Using RNN/LSTM for very long sequences when attention may fit better.
  • Not checking output shape before loss calculation.
