Hyperparameter Tuning & Model Saving

Tune models fairly, save them safely, and make experiments reproducible.

Supports ML Core
OverviewConceptsCodeWorkflowMistakesQuick refPractice

1. Overview

Hyperparameters control how a model learns but are not learned directly from data. Tuning searches for good settings, while saving lets you reuse the trained pipeline consistently.

Mental model

Training learns parameters. Tuning chooses hyperparameters. Validation estimates performance. Saving freezes the chosen pipeline for later inference. Keep those roles separate or evaluation gets cooked.

What you should be able to do after this page

Understand parameter grids, randomized search, cross-validation tuning, nested pipeline parameters, joblib saving, metadata, and reproducibility basics.

3. Hands-on Code

Grid search over a pipeline

Pipeline parameters use step__parameter syntax.

from sklearn.model_selection import GridSearchCV, StratifiedKFold

param_grid = {
    "model__C": [0.1, 1.0, 10.0],
    "model__class_weight": [None, "balanced"],
}
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
search = GridSearchCV(pipe, param_grid=param_grid, scoring="f1", cv=cv, n_jobs=-1)
search.fit(X_train, y_train)
print(search.best_params_)
print(search.best_score_)

Randomized search

Use randomized search when the grid is large or continuous.

from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint

param_dist = {
    "model__n_estimators": randint(100, 800),
    "model__max_depth": randint(2, 20),
}
search = RandomizedSearchCV(pipe, param_dist, n_iter=30, scoring="roc_auc", cv=5, random_state=42, n_jobs=-1)
search.fit(X_train, y_train)

Save and load final pipeline

Save the whole pipeline, not just the model.

import joblib

best_pipeline = search.best_estimator_
joblib.dump(best_pipeline, "model_pipeline.joblib")
loaded = joblib.load("model_pipeline.joblib")
pred = loaded.predict(new_data)

Save metadata

A model without metadata is future-you pain.

import json

model_card = {
    "model_name": "churn_model_v1",
    "metric": "f1",
    "validation_score": float(search.best_score_),
    "best_params": search.best_params_,
}
with open("model_card.json", "w") as f:
    json.dump(model_card, f, indent=2)

4. Practical Workflow

Tune only after a baseline works and a validation scheme is trustworthy.

  1. Choose one primary metric.
  2. Define a realistic CV strategy.
  3. Search a small, sensible parameter space first.
  4. Compare with baseline and inspect variance.
  5. Refit best pipeline on training data.
  6. Evaluate once on the test set.
  7. Save pipeline plus metadata.

5. Common Mistakes

Traps to avoid
  • Huge grids without a time budget.
  • Tuning on test set.
  • Optimizing one metric then reporting another.
  • Saving only model weights while losing preprocessing.
  • Forgetting random_state and package versions.
