1. Overview

Read rows, choose columns, filter conditions, and sort result sets. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

2. Mental Model

How to think about this topic

The core mental model: every command or construct has an input, a transformation, and an output. Strong engineers do not just copy syntax. They predict what should happen, run the smallest possible check, then scale the pattern carefully.

Learning target

After this page, you should be able to explain the purpose of the topic, write the main syntax from scratch, debug common failure modes, and apply it inside a small backend or AI-engineering project.

4. Hands-on Code

Basic SELECT

Start with explicit columns. This keeps output predictable and avoids dragging useless data across the wire.

SELECT id, email, created_at
FROM users
WHERE status = 'active'
ORDER BY created_at DESC
LIMIT 20;

Multiple filters

Parentheses matter when AND and OR live in the same query. SQL is logical, not telepathic.

SELECT id, title, price
FROM products
WHERE stock > 0
  AND (title ILIKE '%python%' OR title ILIKE '%sql%')
ORDER BY price ASC;

NULL-safe filtering

NULL is unknown, so equality checks do not work the way beginners expect.

SELECT id, name, deleted_at
FROM courses
WHERE deleted_at IS NULL;

5. Practical Workflow

  1. Write the business question in plain English.
  2. Choose the base table.
  3. Select only the columns needed.
  4. Add WHERE filters gradually.
  5. Sort and limit after correctness is clear.
  6. Check edge cases: NULL, casing, and empty results.

6. Common Mistakes

Avoid these traps
  • Using SELECT * in production APIs.
  • Using = NULL instead of IS NULL.
  • Forgetting parentheses around mixed AND and OR.
  • Sorting huge results without a limit.
  • Filtering text case-sensitively when user search should be flexible.
