Standard Library: collections, itertools, re & logging

Use Python’s built-in tools for containers, lazy iteration, text patterns, and real debugging output.

1. Mental Model: Batteries Included

Python ships with a large standard library. Before installing another package, check whether Python already has a solid built-in solution. The modules in this handbook are especially useful for intermediate Python: collections, itertools, re, and logging.

2. collections: Better Containers

Counter: count things

from collections import Counter

words = "python python data ai data python".split()
counts = Counter(words)

print(counts)
print(counts.most_common(2))
print(counts["missing"])  # 0, not KeyError

defaultdict: default values for missing keys

from collections import defaultdict

groups = defaultdict(list)
rows = [
    ("A", "Alya"),
    ("B", "Bima"),
    ("A", "Citra"),
]

for group, name in rows:
    groups[group].append(name)

print(dict(groups))

deque: fast queue operations

from collections import deque

queue = deque(["task-1", "task-2"])
queue.append("task-3")

while queue:
    task = queue.popleft()
    print("Processing", task)
Why deque matters

Popping from the front of a list is slow for large lists because elements must shift. deque.popleft() is designed for queues.

3. itertools: Lazy Iterator Building Blocks

itertools functions return iterators, so they are memory efficient. They are excellent for combinations, batching, flattening, and pipeline-style processing.

from itertools import chain, islice, product, combinations

nested = [[1, 2], [3, 4], [5]]
flat = list(chain.from_iterable(nested))
print(flat)

first_three = list(islice(range(100), 3))
print(first_three)

pairs = list(product(["small", "large"], ["red", "blue"]))
print(pairs)

combos = list(combinations(["A", "B", "C"], 2))
print(combos)

groupby: group consecutive items

groupby groups consecutive matching keys. This means you usually sort first. If you skip sorting, you may get multiple groups for the same key.

from itertools import groupby
from operator import itemgetter

rows = [
    {"class": "A", "name": "Alya"},
    {"class": "B", "name": "Bima"},
    {"class": "A", "name": "Citra"},
]

rows = sorted(rows, key=itemgetter("class"))
for class_name, group in groupby(rows, key=itemgetter("class")):
    print(class_name, list(group))

4. re: Regular Expressions

Regular expressions are patterns for matching text. They are useful for validation, extraction, cleaning, and search. They are also very good at becoming unreadable if you try to solve every text problem with one monster pattern. Keep regex small and named.

PatternMeaning
\dDigit
\wWord character
\sWhitespace
+One or more
*Zero or more
?Zero or one
[]Character class
()Capture group
import re

text = "Contact: galih@example.com or admin@test.org"
pattern = r"[\w.-]+@[\w.-]+\.\w+"

emails = re.findall(pattern, text)
print(emails)

cleaned = re.sub(r"\s+", " ", "too    many\nspaces")
print(cleaned)

Compiled patterns and groups

import re

log = "2026-06-21 ERROR payment failed"
pattern = re.compile(r"(?P<date>\d{4}-\d{2}-\d{2}) (?P<level>\w+) (?P<message>.+)")

match = pattern.search(log)
if match:
    print(match.group("date"))
    print(match.group("level"))
    print(match.group("message"))

5. logging: Print Is Not a Monitoring System

print() is fine for beginner debugging. logging is for real programs. It gives levels, timestamps, modules, formatting, and the option to write to files or external systems.

LevelUse when
DEBUGDetailed diagnostic information
INFONormal important events
WARNINGSomething unexpected but program continues
ERROROperation failed
CRITICALProgram may not continue safely
import logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(name)s - %(message)s",
)

logger = logging.getLogger(__name__)

logger.info("Starting data import")
try:
    value = 10 / 0
except ZeroDivisionError:
    logger.exception("Import failed")  # includes traceback

6. Other Standard Library Modules Worth Knowing

  • pathlib for clean filesystem paths.
  • json for reading and writing JSON.
  • csv for CSV files without pandas.
  • datetime for dates, times, and durations.
  • subprocess for running external commands carefully.
  • argparse for command-line interfaces.
from pathlib import Path
import json

path = Path("config.json")
config = {"debug": True, "model": "baseline"}
path.write_text(json.dumps(config, indent=2), encoding="utf-8")

loaded = json.loads(path.read_text(encoding="utf-8"))
print(loaded["model"])

7. Common Mistakes

  • Installing third-party packages for things the standard library already handles.
  • Using regex for nested structured data like HTML or JSON when a parser is better.
  • Using groupby without sorting first.
  • Using print everywhere instead of logging in reusable code.
  • Catching logging errors silently instead of recording tracebacks with logger.exception.
