Transformer Architecture & Attention

Understand attention, tokens, positional encoding, and why transformers dominate NLP.

1. Overview

Transformers power modern NLP and LLMs. Their key idea is attention: each token can look at other tokens and decide what context matters. This makes them more parallelizable and context-aware than classic recurrent models.

Mental model

A transformer turns tokens into vectors, mixes information through self-attention, applies feed-forward layers, and repeats this many times. Attention is weighted information routing.

3. Hands-on Code

Scaled dot-product attention

This is the core attention operation in miniature.

import torch
import torch.nn.functional as F

Q = torch.randn(2, 4, 8)  # batch, tokens, dim
K = torch.randn(2, 4, 8)
V = torch.randn(2, 4, 8)

d_k = Q.size(-1)
scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)
weights = F.softmax(scores, dim=-1)
context = weights @ V
print(context.shape)

Causal mask

Decoder-only LLMs should not attend to future tokens during training.

seq_len = 5
mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1).bool()
scores = torch.randn(seq_len, seq_len)
scores = scores.masked_fill(mask, float("-inf"))
weights = torch.softmax(scores, dim=-1)
print(weights)

Transformer block skeleton

Real blocks include normalization, residuals, attention, and feed-forward networks.

from torch import nn

class TinyBlock(nn.Module):
    def __init__(self, dim, heads):
        super().__init__()
        self.attn = nn.MultiheadAttention(dim, heads, batch_first=True)
        self.ff = nn.Sequential(nn.Linear(dim, 4 * dim), nn.GELU(), nn.Linear(4 * dim, dim))
        self.norm1 = nn.LayerNorm(dim)
        self.norm2 = nn.LayerNorm(dim)

    def forward(self, x, attn_mask=None):
        attn_out, _ = self.attn(x, x, x, attn_mask=attn_mask)
        x = self.norm1(x + attn_out)
        x = self.norm2(x + self.ff(x))
        return x

4. Practical Workflow

Use transformer concepts to debug model behavior: tokenization issues, sequence length limits, masking mistakes, and attention cost.

  1. Tokenize text into IDs.
  2. Convert token IDs into embeddings.
  3. Add or encode position information.
  4. Apply self-attention to mix context.
  5. Use masks for padding or causal prediction.
  6. Stack blocks and train with task-specific objective.

5. Common Mistakes

Traps to avoid
  • Thinking attention is automatically explanation. It is useful signal, not guaranteed causality.
  • Ignoring tokenization, causing weird splits and inflated sequence length.
  • Forgetting attention masks for padding.
  • Confusing encoder-only, decoder-only, and encoder-decoder models.
  • Underestimating quadratic attention cost with long sequences.
