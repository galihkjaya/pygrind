Functional Tools: lambda, map, filter & functools

Use Python functions as values to build clean transformations, callbacks, and reusable utilities.

1. Mental Model: Functions as Values

Functional programming treats functions as values. You can store a function in a variable, pass it into another function, return it from a function, and compose small transformations. Python is not a purely functional language, but these tools are extremely useful for clean data pipelines and reusable logic.

def shout(text):
    return text.upper() + "!"

formatter = shout
print(formatter("hello"))

def apply_to_each(items, fn):
    result = []
    for item in items:
        result.append(fn(item))
    return result

print(apply_to_each(["a", "b"], shout))

2. lambda: Small Anonymous Functions

lambda creates a small one-expression function. It is best for short callbacks, sorting keys, and quick transformations. If your lambda becomes hard to read, write a normal def. Tidak semua harus dibuat aesthetic satu baris.

users = [
    {"name": "Alya", "score": 88},
    {"name": "Bima", "score": 95},
    {"name": "Citra", "score": 80},
]

by_score = sorted(users, key=lambda user: user["score"], reverse=True)
print(by_score)

names = list(map(lambda user: user["name"], users))
print(names)
Lambda limit

A lambda can contain only one expression. No assignment, no multi-line logic, no statements. If logic needs a name, tests, or explanation, use def.

3. map(), filter(), and Comprehension Tradeoffs

map() transforms each item. filter() keeps items that pass a condition. In modern Python, comprehensions are often more readable, but map and filter are still useful when you already have named functions.

numbers = [1, 2, 3, 4, 5]

# map: transform
squares = list(map(lambda n: n * n, numbers))

# filter: keep
odd_numbers = list(filter(lambda n: n % 2 == 1, numbers))

# Often more readable
squares_v2 = [n * n for n in numbers]
odd_numbers_v2 = [n for n in numbers if n % 2 == 1]

print(squares)
print(odd_numbers)

4. functools.reduce(): Combine Many Values Into One

reduce() repeatedly combines values into one result. It is powerful, but do not use it just to look clever. For common tasks, sum, min, max, any, and all are clearer.

from functools import reduce

numbers = [1, 2, 3, 4]
product = reduce(lambda acc, n: acc * n, numbers, 1)
print(product)

# Usually clearer for summing
print(sum(numbers))

5. functools.partial(): Pre-fill Arguments

partial() creates a new function with some arguments already filled in. It is useful for callbacks, dependency injection, and configuration-heavy functions.

from functools import partial

def multiply(a, b):
    return a * b

double = partial(multiply, 2)
triple = partial(multiply, 3)

print(double(10))
print(triple(10))

6. functools.lru_cache(): Memoization

lru_cache remembers previous function results. It is perfect for expensive pure functions where the same input should always produce the same output. It is not good for functions that depend on time, random values, files, databases, or external API state.

from functools import lru_cache

@lru_cache(maxsize=128)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(30))
print(fibonacci.cache_info())
Cache purity rule

Do not cache functions that return different results for the same input. Caching API responses or database queries can be okay only if you understand staleness.

7. operator Module and Key Functions

The operator module gives function versions of common operations. itemgetter and attrgetter are useful for sorting and extracting fields.

from operator import itemgetter

rows = [
    {"name": "Alya", "score": 88},
    {"name": "Bima", "score": 95},
    {"name": "Citra", "score": 80},
]

rows_by_score = sorted(rows, key=itemgetter("score"), reverse=True)
print(rows_by_score)

8. Closures: Functions That Remember

A closure is a function that remembers variables from the scope where it was created. This is the foundation behind many decorators and factory functions.

def make_prefixer(prefix):
    def add_prefix(text):
        return f"{prefix}{text}"
    return add_prefix

add_error = make_prefixer("ERROR: ")
print(add_error("File not found"))

9. Common Mistakes

  • Using lambda for complex logic that deserves a named function.
  • Forgetting that map and filter return iterators in Python 3.
  • Overusing reduce when a loop or built-in aggregator is clearer.
  • Caching functions with side effects or external state.
  • Making functional code so abstract that future-you needs a decoder ring.
