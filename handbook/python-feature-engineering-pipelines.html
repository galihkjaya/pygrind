Feature Engineering & Pipelines

Make raw data model-ready without leaking, drifting, or crying into your CSV.

1. Overview

Feature engineering turns raw columns into useful signals. Pipelines ensure the exact same transformations happen during training, validation, and inference.

Mental model

A feature pipeline is a contract: raw input goes in, numeric model-ready matrix comes out. Every imputer, scaler, encoder, and custom transformation must be learned only from training data.

What you should be able to do after this page

Understand numeric preprocessing, categorical encoding, missing-value handling, custom transformers, ColumnTransformer, and full modeling pipelines.

3. Hands-on Code

ColumnTransformer for mixed data

Numeric and categorical columns need different preprocessing.

from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.impute import SimpleImputer
from sklearn.linear_model import LogisticRegression

num_cols = ["age", "income", "tenure"]
cat_cols = ["city", "plan_type"]

num_pipe = Pipeline([("impute", SimpleImputer(strategy="median")), ("scale", StandardScaler())])
cat_pipe = Pipeline([("impute", SimpleImputer(strategy="most_frequent")), ("onehot", OneHotEncoder(handle_unknown="ignore"))])
preprocess = ColumnTransformer([("num", num_pipe, num_cols), ("cat", cat_pipe, cat_cols)])
pipe = Pipeline([("preprocess", preprocess), ("model", LogisticRegression(max_iter=1000))])
pipe.fit(X_train, y_train)

Custom feature

Keep custom transformations deterministic and inference-safe.

import numpy as np
from sklearn.preprocessing import FunctionTransformer

def add_ratio_features(X):
    X = X.copy()
    X["spend_per_month"] = X["total_spend"] / np.maximum(X["tenure_months"], 1)
    return X

feature_adder = FunctionTransformer(add_ratio_features, validate=False)

Feature names

Feature names are useful for debugging and explanation.

pipe.fit(X_train, y_train)
feature_names = pipe.named_steps["preprocess"].get_feature_names_out()
print(feature_names[:20])

4. Practical Workflow

Feature engineering is hypothesis-driven. Each feature should have a reason to exist, not just because “maybe the model likes it”.

  1. Separate features from target.
  2. Split before fitting transformations.
  3. Classify columns into numeric, categorical, datetime, text, or special.
  4. Build preprocessing pipelines per column type.
  5. Combine with ColumnTransformer.
  6. Validate transformed shape and missing values.
  7. Save preprocessing and model together.

5. Common Mistakes

Traps to avoid
  • Fitting encoders or imputers on the full dataset.
  • Using label encoding for nominal categories with linear models.
  • Forgetting handle_unknown="ignore" for OneHotEncoder.
  • Creating features from future information.
  • Letting train and inference preprocessing differ.
