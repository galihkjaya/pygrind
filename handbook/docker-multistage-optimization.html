1. Overview

Build smaller, safer images with better layer caching. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

2. Mental Model

How to think about this topic

The core mental model: every command or construct has an input, a transformation, and an output. Strong engineers do not just copy syntax. They predict what should happen, run the smallest possible check, then scale the pattern carefully.

Learning target

After this page, you should be able to explain the purpose of the topic, write the main syntax from scratch, debug common failure modes, and apply it inside a small backend or AI-engineering project.

4. Hands-on Code

Multi-stage concept

Build artifacts in one stage, copy only needed output to runtime.

FROM node:20 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html

Python wheel builder

Useful when dependencies need build tools.

FROM python:3.12-slim AS builder
WORKDIR /wheels
COPY requirements.txt .
RUN pip wheel --no-cache-dir --wheel-dir /wheels -r requirements.txt

FROM python:3.12-slim
WORKDIR /app
COPY --from=builder /wheels /wheels
RUN pip install --no-cache-dir /wheels/*
COPY . .

5. Practical Workflow

  1. Measure image size before optimizing.
  2. Move build tools into builder stage.
  3. Copy only required artifacts into runtime.
  4. Order layers from least-changing to most-changing.
  5. Use slim/alpine carefully after checking compatibility.
  6. Rebuild and compare size plus startup behavior.

6. Common Mistakes

Avoid these traps
  • Optimizing before the app even works.
  • Using alpine with incompatible binary wheels without checking.
  • Leaving compilers/build tools in runtime image.
  • Breaking cache by copying source too early.
  • Chasing tiny image size while destroying debuggability.
