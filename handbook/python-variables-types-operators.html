1. Variables

A variable is a named container for a value. In Python, you don't declare types — you just assign, and Python figures it out.

# Assignment — just use the equals sign
name = "Andi"
age = 25
height = 1.75
is_student = True

# Multiple assignment at once
x, y, z = 1, 2, 3

# Same value to multiple variables
a = b = c = 0

# Swap without a temp variable (Python magic!)
x, y = y, x

How Python stores variables

Python variables are not boxes containing values — they're labels pointing to objects in memory. When you write x = 5, Python creates an integer object 5 somewhere in memory, then makes the name x point to it.

x = 5
y = x        # y points to the SAME object as x
print(id(x) == id(y))   # True — same memory address

y = 10       # Now y points to a new object
print(x)    # Still 5 — x is unaffected

Naming Rules

  • Must start with a letter or underscore: name, _hidden
  • Can contain letters, digits, underscores: user_id_2
  • Case-sensitive: Name and name are different
  • Cannot be a keyword: for, if, class, etc.

2. Data Types

Python has a handful of built-in types you'll use constantly.

The Core Types

# int — whole numbers, unlimited size
age = 25
big = 10_000_000       # underscores for readability
binary = 0b1010        # binary literal → 10
hexval = 0xFF          # hex literal → 255

# float — decimal numbers (64-bit)
price = 9.99
sci = 1.5e3            # scientific → 1500.0

# str — text, immutable, Unicode
greeting = "Hello"
multi = """Line one
Line two"""

# bool — True or False (subclass of int!)
logged_in = True
print(True + True)        # 2 — bool is int under the hood

# NoneType — the absence of a value
result = None
print(type(result))       # <class 'NoneType'>

Checking Types

x = 42

# type() returns the exact type
print(type(x))            # <class 'int'>
print(type(x) == int)    # True

# isinstance() is better — handles inheritance
print(isinstance(x, int))     # True
print(isinstance(True, int))  # True — bool IS an int

Mutable vs Immutable

Immutable ❌ (can't change)

int, float, str, bool, tuple, frozenset

s = "hello"
s[0] = "H"  # TypeError!
# Must create new string
s = "H" + s[1:]

Mutable ✅ (can change)

list, dict, set, bytearray

lst = [1, 2, 3]
lst[0] = 99  # Works!
print(lst)    # [99, 2, 3]
⚠️ Mutable Default Argument Trap This is one of Python's most infamous gotchas. Never use mutable objects as default arguments — it will bite you later (covered more in Functions).

3. Type Casting

Converting between types is called casting or type conversion. Python is strongly-typed — it won't silently mix types for you.

# Explicit casting
int("42")         # → 42
int(3.9)          # → 3  (truncates, doesn't round!)
float("3.14")     # → 3.14
float(5)          # → 5.0
str(100)          # → "100"
bool(0)           # → False
bool("")          # → False
bool("hello")     # → True
list("abc")       # → ['a', 'b', 'c']
tuple([1,2,3])   # → (1, 2, 3)

Truthy and Falsy Values

Every Python value can be evaluated as True or False in a boolean context.

Falsy values

False
None
0      # int zero
0.0    # float zero
""     # empty string
[]     # empty list
{}     # empty dict
()     # empty tuple
set()  # empty set

Everything else is truthy

True
1, -1, 42
"hello"
[0]         # list with 0 in it!
{"a": None} # dict with None val
lambda: None # any function
# Practical use: check if something "exists"
name = ""
if not name:
    print("Name is empty")

data = []
if data:
    print("Has data")
else:
    print("Empty!")   # This runs

4. Operators

Arithmetic Operators

a, b = 17, 5

a + b    # 22  — addition
a - b    # 12  — subtraction
a * b    # 85  — multiplication
a / b    # 3.4 — division (ALWAYS returns float)
a // b   # 3   — floor division (integer result)
a % b    # 2   — modulo (remainder)
a ** b   # 1419857 — exponentiation (17^5)
💡 // and % are related a == (a // b) * b + (a % b) always holds. So 17 == 3*5 + 2 ✓

Comparison Operators

x = 10

x == 10   # True  — equal value
x != 10   # False — not equal
x > 5    # True  — greater than
x < 5    # False — less than
x >= 10  # True  — greater or equal
x <= 9   # False — less or equal

# Chaining comparisons (very Pythonic)
1 < x < 100   # True — equivalent to (1 < x) and (x < 100)
0 <= x <= 10  # True

Logical Operators

# and, or, not
True and False   # False — both must be True
True or False    # True  — at least one True
not True         # False

# Short-circuit evaluation
# 'and' stops at first False
False and print("never runs")  # print() not called

# 'or' stops at first True
True or print("never runs")   # print() not called

# 'and'/'or' return the actual value (not just True/False)
"hello" or "world"   # "hello" — first truthy
"" or "default"      # "default" — first is falsy
0 and 99             # 0 — short-circuits at 0
5 and 99             # 99 — 5 is truthy, returns last
✅ Practical pattern: default values 
username = user_input or "Guest"
# If user_input is empty/None, use "Guest"

Identity vs Equality

# == checks VALUE equality
# is checks IDENTITY (same object in memory)

a = [1, 2, 3]
b = [1, 2, 3]
c = a

a == b   # True  — same values
a is b   # False — different objects
a is c   # True  — c points to same object as a

# Always use 'is' for None, True, False
x = None
if x is None:      # ✅ correct
    print("No value")
if x == None:      # ⚠️ works but not Pythonic
    print("No value")

Membership Operators

fruits = ["apple", "banana", "mango"]

"apple" in fruits      # True
"grape" not in fruits   # True

# Also works on strings, dicts, sets
"ell" in "hello"        # True — substring check
"key" in {"key": 1}     # True — checks dict KEYS

Bitwise Operators

# Less common but important for ML/AI (flags, masks)
a = 0b1010   # 10
b = 0b1100   # 12

a & b    # 0b1000 = 8  — AND
a | b    # 0b1110 = 14 — OR
a ^ b    # 0b0110 = 6  — XOR
~a       # -11         — NOT (bitwise complement)
a << 1   # 20          — left shift (multiply by 2)
a >> 1   # 5           — right shift (divide by 2)

Assignment Operators

x = 10
x += 5    # x = x + 5  → 15
x -= 3    # x = x - 3  → 12
x *= 2    # x = x * 2  → 24
x /= 4    # x = x / 4  → 6.0 (float!)
x //= 2   # x = x // 2 → 3.0
x **= 3   # x = x ** 3 → 27.0
x %= 5    # x = x % 5  → 2.0

# Walrus operator := (Python 3.8+) — assign AND use in expression
data = [1, 2, 3, 4, 5]
if (n := len(data)) > 3:
    print(f"List has {n} elements, too long")

# Common precedence surprises
2 + 3 * 4        # 14, not 20  (* before +)
2 ** 3 ** 2      # 512, not 64 (right-associative: 3**2=9, 2**9=512)
not True or True # True (not binds tighter: (not True) or True)

# When in doubt, use parentheses — clarity > cleverness
result = (2 + 3) * 4   # Explicitly 20

f-strings (Essential!)

name = "Andi"
score = 98.5
level = "Advanced"

# Basic f-string
print(f"Hello, {name}!")         # Hello, Andi!

# Expressions inside {}
print(f"2 + 2 = {2 + 2}")        # 2 + 2 = 4

# Format specifiers
print(f"Score: {score:.1f}%")     # Score: 98.5%
print(f"{1000000:,}")            # 1,000,000
print(f"{score:.2f}")            # 98.50
print(f"{42:05d}")             # 00042 (zero-padded)
print(f"{'hi':>10}")            #         hi (right-aligned)

# Debug with = (Python 3.8+)
x = 42
print(f"{x=}")                  # x=42

6. Common Mistakes

❌ Using == for None 
# Bad
if x == None: ...
# Good
if x is None: ...
❌ Integer division losing the decimal 
result = 7 / 2    # 3.5 ✅ (Python 3)
result = 7 // 2   # 3   ⚠️ floor division
❌ Float precision issues 
print(0.1 + 0.2)         # 0.30000000000000004 !
print(0.1 + 0.2 == 0.3)  # False !

# Fix: use round() or decimal module
round(0.1 + 0.2, 10) == 0.3   # True
from decimal import Decimal
Decimal("0.1") + Decimal("0.2")  # Exact
❌ String + number concatenation 
age = 25
print("Age: " + age)       # TypeError!
print("Age: " + str(age))  # ✅
print(f"Age: {age}")        # ✅ better
