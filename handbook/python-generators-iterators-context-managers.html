Generators, Iterators & Context Managers

Write memory-efficient loops, lazy pipelines, and safe setup-cleanup blocks.

1. Mental Model: Pull Values One at a Time

Iteration is the protocol that lets Python pull values one by one. A list gives you all values already stored in memory. An iterator gives you the next value only when asked. A generator is a convenient way to create an iterator using yield.

Why this matters

Iterators make pipelines memory efficient. Instead of loading a massive file into a list, you can process one line at a time. That is the difference between “works on my laptop” and “my laptop is now a jet engine”.

2. Iterables and Iterators

An iterable knows how to create an iterator. An iterator remembers where it is. When a for loop runs, Python internally calls iter() once and next() repeatedly until StopIteration.

class Countdown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value

for n in Countdown(3):
    print(n)
Iterator exhaustion

Many iterators can be consumed only once. After you loop through them, they are empty. If you need to reuse values, convert to a list intentionally, but only when the data is small enough.

items = iter(["a", "b", "c"])
print(list(items))  # consumes the iterator
print(list(items))  # now empty

3. Generator Functions and yield

A generator function looks like a normal function, but it uses yield instead of return. Each time Python reaches yield, it pauses the function and remembers its local state. The next next() resumes from that same point.

def read_clean_lines(path):
    with open(path, "r", encoding="utf-8") as file:
        for line in file:
            line = line.strip()
            if line:
                yield line

for line in read_clean_lines("notes.txt"):
    print(line.upper())

yield vs return

return ends the function and gives one result. yield pauses the function and can produce many results over time. Use yield when the output is a stream.

4. Generator Expressions

A generator expression is like a list comprehension, but it uses parentheses and does not build the full list immediately. It is lazy.

numbers = range(1_000_000)

# Builds a full list in memory
squares_list = [n * n for n in numbers]

# Produces values only when requested
squares_gen = (n * n for n in numbers)

print(next(squares_gen))
print(next(squares_gen))
Good use case

Generator expressions are great inside aggregators like sum, any, all, and max when you do not need to store intermediate results.

scores = [85, 92, 70, 100]
passed_count = sum(1 for score in scores if score >= 75)
all_valid = all(0 <= score <= 100 for score in scores)
print(passed_count, all_valid)

5. Streaming Pipelines

Generators compose nicely. You can build small steps that each do one thing: read rows, clean rows, parse rows, filter rows, transform rows. This is common in ETL, log processing, and ML preprocessing.

def tokenize(lines):
    for line in lines:
        yield line.lower().split()

def remove_short_words(token_rows, min_len=3):
    for tokens in token_rows:
        yield [token for token in tokens if len(token) >= min_len]

lines = ["Python is fun", "AI needs clean data"]
pipeline = remove_short_words(tokenize(lines))

for tokens in pipeline:
    print(tokens)

6. Context Managers and with

A context manager handles setup and cleanup. The most common one is open(). It closes the file even if an error happens inside the block. This is safer than manually calling close().

with open("data.txt", "r", encoding="utf-8") as file:
    text = file.read()

# file is already closed here

You can create your own context manager using a class with __enter__ and __exit__.

class Timer:
    def __enter__(self):
        import time
        self.start = time.perf_counter()
        return self

    def __exit__(self, exc_type, exc_value, traceback):
        import time
        self.end = time.perf_counter()
        self.elapsed = self.end - self.start
        print(f"Elapsed: {self.elapsed:.4f}s")
        return False  # do not suppress exceptions

with Timer():
    total = sum(range(1_000_000))

7. contextlib: Context Managers Without a Class

For simple context managers, contextlib.contextmanager lets you write one generator function. Code before yield is setup. Code after yield is cleanup.

from contextlib import contextmanager

@contextmanager
def temporary_setting(settings, key, value):
    old_value = settings.get(key)
    settings[key] = value
    try:
        yield settings
    finally:
        if old_value is None:
            settings.pop(key, None)
        else:
            settings[key] = old_value

config = {"debug": False}
with temporary_setting(config, "debug", True):
    print(config["debug"])
print(config["debug"])

8. Common Mistakes

  • Calling next() too many times without handling StopIteration.
  • Expecting a generator to restart after it has been consumed.
  • Turning huge generators into lists too early and losing the memory benefit.
  • Putting too much logic inside one giant generator instead of composing smaller ones.
  • Forgetting that __exit__ returning True suppresses exceptions.
