1. Mental Model: Containers for Different Jobs

Data structures are how you organize values so your program can read, update, search, and transform them cleanly. In Python, the four foundation containers are list, dict, set, and tuple. They look simple, but they are the backbone of real Python work: API responses, ML datasets, config files, logs, features, and user state.

StructureBest mental modelOrdered?Mutable?Allows duplicates?Typical use
listA sequence of itemsYesYesYesRows, tasks, scores, tokens
dictKey → value mappingYes, insertion orderYesKeys uniqueRecords, JSON, lookup tables
setA bag of unique valuesNo reliable positionYesNoDeduplication, membership tests
tupleA fixed-size recordYesNoYesCoordinates, immutable pairs, function returns
Core idea Do not choose a data structure based on vibe. Choose based on the operation you need most: preserve order, update values, prevent duplicates, look up by key, or protect from mutation.

2. Lists: Ordered, Mutable Sequences

A list is the default container when you have multiple items in order. It is perfect when position matters, when duplicates are valid, and when you expect to add, remove, sort, or transform items.

# Creating lists
scores = [80, 92, 75, 90]
names = ["Alya", "Bima", "Citra"]
mixed = ["Python", 3.12, True]
empty = []

# Accessing by index, starting at 0
first_score = scores[0]
last_score = scores[-1]

# Slicing: list[start:stop:step]
top_two = scores[:2]
without_first = scores[1:]
reversed_scores = scores[::-1]

print(first_score)
print(top_two)
print(reversed_scores)

List mutation: changing the same object

tasks = ["read", "practice"]

tasks.append("review")          # add to end
tasks.insert(1, "watch video")  # insert at index 1
tasks.remove("read")            # remove by value
done = tasks.pop()               # remove and return last item

tasks[0] = "solve problem"       # replace item by index

print(tasks)
print(done)

When a list is the right choice

  • You care about order: first, second, last, sorted, reversed.
  • You need duplicates: two students can have the same score.
  • You will loop through all items and process them one by one.
  • You need a growable collection, like user inputs or scraped rows.

Common list patterns

numbers = [5, 2, 9, 1, 5]

# Loop over values
for number in numbers:
    print(number)

# Loop with index and value
for index, number in enumerate(numbers):
    print(index, number)

# Sorting without changing original
sorted_numbers = sorted(numbers)

# Sorting in-place
numbers.sort()

# Check membership
if 9 in numbers:
    print("Found 9")

# Aggregate operations
print(len(numbers))
print(sum(numbers))
print(min(numbers), max(numbers))

Nested lists: tables and matrices

A nested list is a list that contains other lists. In data work, this often represents a table, matrix, grid, or mini dataset.

matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9],
]

# Access row 1, column 2
value = matrix[1][2]

# Loop through every cell
for row in matrix:
    for cell in row:
        print(cell)

# Add a new row
matrix.append([10, 11, 12])
❌ Trap: aliasing nested lists

Do not create repeated inner lists with multiplication when you plan to mutate them. All rows may point to the same inner list. Python is not bugged, your reference is just doing parkour.

bad_grid = [[0] * 3] * 3
bad_grid[0][0] = 1
print(bad_grid)  # All rows changed at column 0

# Safer pattern
good_grid = [[0 for _ in range(3)] for _ in range(3)]
good_grid[0][0] = 1
print(good_grid)

3. Tuples: Ordered, Immutable Records

A tuple is like a list that should not be changed. Use it when the position has meaning and the shape of the data is fixed: coordinate pairs, RGB colors, database rows, or multiple return values.

point = (3, 4)
name_age = ("Nadia", 21)
empty_tuple = ()
single_item_tuple = ("Python",)  # comma matters

x = point[0]
y = point[1]

# Tuple unpacking
name, age = name_age

print(x, y)
print(name, age)

Tuple unpacking makes code cleaner

student = ("Raka", 88, "Data Science")
name, score, major = student

print(f"{name} scored {score} in {major}")

# Swap values without temporary variable
a = 10
b = 20
a, b = b, a

# Returning multiple values from a function
def summarize(numbers):
    return min(numbers), max(numbers), sum(numbers) / len(numbers)

lowest, highest, average = summarize([10, 20, 30])
⚠️ Immutable does not always mean deeply immutable A tuple cannot replace its items, but if an item inside it is mutable, that inner object can still change. 
data = ([1, 2], "label")
# data[0] = [9, 9]     # TypeError: tuple item assignment

data[0].append(3)      # Allowed, because the inner list is mutable
print(data)

4. Dictionaries: Key → Value Lookup

A dict stores pairs of keys and values. This is the structure you use when you want to ask: “given this key, what is the value?” Dicts are everywhere in real Python because JSON objects become dicts.

student = {
    "name": "Dewi",
    "major": "Statistics",
    "score": 91,
    "is_active": True,
}

# Access by key
print(student["name"])

# Safer access when key may not exist
print(student.get("email"))
print(student.get("email", "not provided"))

# Add or update
student["score"] = 95
student["email"] = "dewi@example.com"

# Remove
removed_email = student.pop("email")

print(student)

Looping through dictionaries

profile = {
    "username": "galih15",
    "level": "beginner",
    "goal": "learn backend",
}

for key in profile:
    print(key)

for value in profile.values():
    print(value)

for key, value in profile.items():
    print(f"{key}: {value}")

Counting and grouping with dict

One of the most useful beginner-to-intermediate patterns is using a dict as a counter or grouping tool.

words = ["python", "data", "python", "ai", "data", "python"]
counts = {}

for word in words:
    counts[word] = counts.get(word, 0) + 1

print(counts)

# Group scores by class
rows = [
    {"class": "A", "score": 80},
    {"class": "B", "score": 75},
    {"class": "A", "score": 90},
]

groups = {}
for row in rows:
    class_name = row["class"]
    groups.setdefault(class_name, []).append(row["score"])

print(groups)
dict keys must be hashable Strings, numbers, booleans, and tuples of immutable values can be keys. Lists and dicts cannot be keys because they can change.

5. Sets: Unique Values and Fast Membership

A set stores unique values. Use it when you care whether something exists, not where it is located. Membership checks are usually much faster than checking inside a long list.

tags = {"python", "ml", "data", "python"}
print(tags)  # duplicate removed

# Create empty set
empty_set = set()

# Add and remove
tags.add("backend")
tags.discard("ml")      # no error if missing
# tags.remove("ml")     # error if missing

# Membership
if "python" in tags:
    print("Python tag exists")

Set operations

frontend = {"html", "css", "javascript", "react"}
backend = {"python", "sql", "javascript", "api"}

print(frontend | backend)  # union: everything
print(frontend & backend)  # intersection: common items
print(frontend - backend)  # difference: frontend only
print(frontend ^ backend)  # symmetric difference: not shared

Practical use: deduplicate while keeping order

A normal set removes duplicates but does not preserve meaningful position. To deduplicate a list while keeping the first appearance, combine a set with a list.

names = ["Ayu", "Bima", "Ayu", "Citra", "Bima"]
seen = set()
unique_names = []

for name in names:
    if name not in seen:
        unique_names.append(name)
        seen.add(name)

print(unique_names)

7. Real-World Patterns

Pattern A: list of dicts for tabular records

students = [
    {"name": "Alya", "score": 88, "passed": True},
    {"name": "Bima", "score": 74, "passed": True},
    {"name": "Citra", "score": 59, "passed": False},
]

passed_names = []
for student in students:
    if student["passed"]:
        passed_names.append(student["name"])

print(passed_names)

Pattern B: dict of lists for grouping

transactions = [
    {"category": "food", "amount": 25000},
    {"category": "transport", "amount": 12000},
    {"category": "food", "amount": 18000},
]

by_category = {}
for trx in transactions:
    category = trx["category"]
    by_category.setdefault(category, []).append(trx["amount"])

print(by_category)

Pattern C: set for validation

allowed_roles = {"admin", "editor", "viewer"}
user_role = "editor"

if user_role not in allowed_roles:
    print("Invalid role")
else:
    print("Role accepted")

9. Common Mistakes

❌ Using a list when you need lookup by ID

If you keep searching a list to find a user by ID, a dict is probably cleaner and faster.

users = [
    {"id": 1, "name": "Alya"},
    {"id": 2, "name": "Bima"},
]

# Better lookup structure
users_by_id = {
    1: {"id": 1, "name": "Alya"},
    2: {"id": 2, "name": "Bima"},
}

print(users_by_id[2])
❌ Confusing copy and alias 
a = [1, 2, 3]
b = a          # alias: same list object
b.append(4)
print(a)       # also changed

c = a.copy()   # shallow copy: new outer list
c.append(5)
print(a)
print(c)
⚠️ Updating a dict while looping over it

Changing dict size while iterating over it can raise errors. Loop over list(my_dict) or create a new dict.

scores = {"Alya": 90, "Bima": 45, "Citra": 70}

for name in list(scores):
    if scores[name] < 50:
        del scores[name]

print(scores)
