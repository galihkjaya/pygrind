1. Overview

NumPy is the base layer for scientific Python. Pandas, scikit-learn, PyTorch, and many ML tools share the same core idea: store numeric data in compact arrays and operate on whole arrays instead of slow Python loops.

Mental model

A NumPy array is a typed rectangular block of numbers. Always ask three questions: what is the shape, what is the dtype, and which axis am I operating on? Shape bugs are the classic “looks fine but explodes later” villain.

What you should be able to do after this page

Create arrays, inspect shape and dtype, use vectorized math, filter with boolean masks, understand broadcasting, and apply basic linear algebra for ML-style workflows.

3. Hands-on Code

Create arrays and inspect them

Start debugging by printing shape and dtype. Tiny habit, huge payoff.

import numpy as np

x = np.array([10, 20, 30])
X = np.array([[1, 2, 3], [4, 5, 6]])

print(x.shape, x.dtype)
print(X.shape, X.ndim, X.size)
print(X[0, 1])
print(X[:, 0])

Vectorization and masks

Vectorized operations describe what you want; NumPy handles the optimized loop internally.

scores = np.array([55, 72, 90, 64, 88])
passed = scores >= 70
print(scores[passed])
print((scores - scores.mean()) / scores.std())

Broadcasting for scaling

A column vector of statistics can be applied to every row when shapes are compatible.

X = np.array([[170, 65], [160, 50], [180, 80]])
means = X.mean(axis=0)
stds = X.std(axis=0)
X_scaled = (X - means) / stds
print(X_scaled)

Linear algebra basics

ML models often reduce to matrix multiplication, dot products, and solving systems.

w = np.array([0.7, -0.2, 0.1])
x = np.array([2.0, 5.0, 1.0])
print(np.dot(w, x))

A = np.array([[1, 2], [3, 4]])
b = np.array([10, 20])
print(np.linalg.solve(A, b))

4. Practical Workflow

A clean NumPy workflow keeps data numeric, checks shapes early, transforms by axis intentionally, and avoids Python loops unless the operation is truly not vectorizable.

  1. Convert raw lists to arrays after deciding intended shape.
  2. Inspect shape, dtype, min/max, and missing-value strategy.
  3. Use vectorized operations for math.
  4. Use masks for filtering.
  5. Normalize feature columns with axis=0.
  6. Test transformations on tiny arrays before scaling up.

5. Common Mistakes

Traps to avoid
  • Confusing row-wise and column-wise operations.
  • Using Python loops for vectorizable math.
  • Forgetting that shape (n,) and (n, 1) broadcast differently.
  • Accidentally mixing strings with numbers and forcing object dtype.
  • Using manual matrix inverse when np.linalg.solve is safer.
