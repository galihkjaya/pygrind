Text Preprocessing & Classical NLP

TF-IDF, embeddings, and text features before transformers enter the chat.

1. Overview

Classical NLP still matters because it is fast, cheap, interpretable, and often strong for baseline text classification or search. Before using transformers, understand tokenization, normalization, n-grams, TF-IDF, and embeddings.

Mental model

Text is not numeric by default. NLP turns text into features: counts, weights, vectors, or contextual representations. The representation decides what information the model can actually use.

What you should be able to do after this page

Clean text, tokenize, build bag-of-words and TF-IDF features, train a text classifier, understand embeddings, and avoid preprocessing choices that destroy meaning.

3. Hands-on Code

Basic cleaning

Clean text intentionally. Do not delete useful signal blindly.

import re

def clean_text(text):
    text = text.lower().strip()
    text = re.sub(r"https?://\S+", " URL ", text)
    text = re.sub(r"@\w+", " USER ", text)
    text = re.sub(r"\s+", " ", text)
    return text

print(clean_text("Wow!!! check https://example.com @andi"))

TF-IDF classifier

A strong baseline for text classification.

from sklearn.pipeline import Pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report

pipe = Pipeline([
    ("tfidf", TfidfVectorizer(ngram_range=(1, 2), min_df=2, max_features=50000)),
    ("clf", LogisticRegression(max_iter=1000)),
])

pipe.fit(X_train_text, y_train)
pred = pipe.predict(X_test_text)
print(classification_report(y_test, pred))

Inspect important terms

Classical models are more interpretable than most neural models.

vectorizer = pipe.named_steps["tfidf"]
clf = pipe.named_steps["clf"]
feature_names = vectorizer.get_feature_names_out()

coefs = clf.coef_[0]
top_positive = coefs.argsort()[-10:][::-1]
print(feature_names[top_positive])

4. Practical Workflow

Always build a classical NLP baseline before using a large model. It gives you a cheap performance floor and reveals dataset issues quickly.

  1. Define label and text field.
  2. Split data before fitting vectorizer.
  3. Apply minimal cleaning that preserves task signal.
  4. Build TF-IDF plus linear model baseline.
  5. Inspect errors and top features.
  6. Only move to transformers if baseline is insufficient.

5. Common Mistakes

Traps to avoid
  • Removing negation words like “not” and destroying sentiment meaning.
  • Fitting TF-IDF on full data before splitting.
  • Over-cleaning social media text and removing important tokens.
  • Ignoring class imbalance.
  • Jumping to transformers before checking a cheap baseline.
