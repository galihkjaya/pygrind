1. Overview

Automate repeated commands and manage configuration safely. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

2. Mental Model

How to think about this topic

The core mental model: every command or construct has an input, a transformation, and an output. Strong engineers do not just copy syntax. They predict what should happen, run the smallest possible check, then scale the pattern carefully.

Learning target

After this page, you should be able to explain the purpose of the topic, write the main syntax from scratch, debug common failure modes, and apply it inside a small backend or AI-engineering project.

4. Hands-on Code

Script skeleton

Start scripts with strict-ish settings for safer failure.

#!/usr/bin/env bash
set -euo pipefail

echo "Running checks..."
python -m pytest

Arguments

Scripts can receive values from command line.

#!/usr/bin/env bash
NAME=${1:-world}
echo "Hello, $NAME

Environment variables

Use env vars for config, not hardcoded secrets.

export APP_ENV=development
export DATABASE_URL=postgresql://postgres:secret@localhost:5432/app
python app/main.py

5. Practical Workflow

  1. Start with commands you already run manually.
  2. Move them into a .sh file.
  3. Add a shebang and executable permission.
  4. Use arguments for changing values.
  5. Use environment variables for config/secrets.
  6. Check exit codes in CI.

6. Common Mistakes

Avoid these traps
  • Hardcoding secrets in scripts.
  • Forgetting chmod +x.
  • Using unquoted variables and breaking paths with spaces.
  • Ignoring failed commands.
  • Committing .env with real secrets.
