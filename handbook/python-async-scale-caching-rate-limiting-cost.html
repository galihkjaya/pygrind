Async at Scale: Caching, Rate Limiting & Cost Optimization

Make AI apps faster, cheaper, and less likely to catch fire.

Supports AI Production
OverviewConceptsCodeWorkflowMistakesQuick refPractice

1. Overview

AI apps often call slow external services. Async programming, caching, rate limiting, batching, and retries help control latency, cost, and reliability.

Mental model

Scale is controlled concurrency. You want enough parallelism to be fast, but not so much that you exceed provider limits, duplicate work, or burn budget like a finance speedrun.

What you should be able to do after this page

Use async/await for concurrent I/O, apply semaphores, cache repeated calls, implement retries, rate limit requests, batch work, and log cost.

3. Hands-on Code

Concurrent async calls

Use gather for I/O-bound provider calls.

import asyncio

async def call_model(prompt):
    await asyncio.sleep(0.2)  # pretend network call
    return prompt.upper()

async def main():
    prompts = ["a", "b", "c"]
    results = await asyncio.gather(*(call_model(p) for p in prompts))
    print(results)

asyncio.run(main())

Limit concurrency

Semaphores prevent request storms.

import asyncio

sem = asyncio.Semaphore(3)

async def safe_call(prompt):
    async with sem:
        return await call_model(prompt)

results = await asyncio.gather(*(safe_call(p) for p in prompts))

Simple cache key

Cache deterministic calls when possible.

import hashlib, json, time

cache = {}

def make_key(payload):
    raw = json.dumps(payload, sort_keys=True).encode()
    return hashlib.sha256(raw).hexdigest()

async def cached_call(payload, ttl=300):
    key = make_key(payload)
    now = time.time()
    if key in cache and now - cache[key]["time"] < ttl:
        return cache[key]["value"]
    value = await call_model(payload["prompt"])
    cache[key] = {"time": now, "value": value}
    return value

4. Practical Workflow

Optimize only after measuring. But some design choices, like timeouts and rate limits, should exist from day one.

  1. Add request timeouts.
  2. Use async for external I/O.
  3. Limit concurrency with semaphores.
  4. Cache repeated deterministic calls.
  5. Retry transient 429/5xx failures with backoff.
  6. Batch small requests when provider supports it.
  7. Log tokens, latency, errors, and estimated cost.

5. Common Mistakes

Traps to avoid
  • Unlimited concurrency. Congrats, you invented self-DDoS.
  • Caching outputs for prompts that include private or changing data without policy.
  • Retrying non-retryable errors forever.
  • No timeout, causing hanging requests.
  • Optimizing cost without monitoring quality regression.
