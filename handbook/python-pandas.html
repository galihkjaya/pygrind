Pandas: DataFrames, Filtering, Grouping & Merging

Work with tabular data like a serious analyst, not like a spreadsheet gremlin.

1. Overview

Pandas is the main Python tool for loading, cleaning, transforming, summarizing, and joining tabular data. If NumPy is the numeric engine, Pandas is the data-wrangling control room.

Mental model

A DataFrame is a labeled table. Each column is a Series with a name, dtype, and index. Most tasks are one of five operations: select rows, select columns, transform values, group rows, or join tables.

What you should be able to do after this page

Load data, inspect structure, filter rows, create columns, handle missing values, group and aggregate, merge tables, and avoid chained-assignment bugs.

3. Hands-on Code

Inspect before modeling

Before modeling, inspect. Bad data loves hiding in plain sight.

import pandas as pd

df = pd.DataFrame({
    "student": ["Alya", "Bima", "Citra", "Doni"],
    "major": ["stats", "cs", "stats", "math"],
    "score": [88, 76, 92, 69],
    "hours": [5, 3, 7, 2],
})

print(df.head())
print(df.info())
print(df.describe(numeric_only=True))

Filtering and feature creation

Use boolean masks for rows and explicit .loc for safe assignment.

passed = df[df["score"] >= 75].copy()
passed["score_per_hour"] = passed["score"] / passed["hours"]

mask = df["major"].eq("stats")
df.loc[mask, "is_stats"] = True
df["is_stats"] = df["is_stats"].fillna(False)
print(passed)

Group and aggregate

Groupby answers questions like average score by major or count users by segment.

summary = (
    df.groupby("major")
      .agg(n_students=("student", "count"), avg_score=("score", "mean"), max_hours=("hours", "max"))
      .reset_index()
)
print(summary)

Merge two tables

Most real datasets are split across tables. Learn joins early or suffer later.

profiles = pd.DataFrame({"student": ["Alya", "Bima", "Citra"], "city": ["Semarang", "Solo", "Tegal"]})
merged = df.merge(profiles, on="student", how="left")
print(merged)

4. Practical Workflow

A reliable Pandas workflow is inspect, clean, transform, summarize, validate, then export. Do not jump directly into modeling with a mystery table.

  1. Load with explicit separator, encoding, date columns, and dtypes when needed.
  2. Inspect shape, info, missing values, duplicates, and basic statistics.
  3. Standardize column names and categories.
  4. Create features using vectorized operations.
  5. Group or merge only after checking keys.
  6. Save a clean intermediate file if cleaning is long.

5. Common Mistakes

Traps to avoid
  • Chained assignment like df[df.x > 0]["y"] = 1. Use .loc.
  • Merging without checking duplicate keys.
  • Using apply for simple vectorized operations.
  • Assuming missing values are random without checking patterns.
  • Letting object/string columns sneak into numeric modeling pipelines.
