1. Overview

PyTorch is a deep learning framework centered on tensors, automatic differentiation, and modular neural network layers. It gives you control while handling gradients and GPU acceleration.

Mental model

PyTorch tensors are NumPy-like arrays that can track computation history. If requires_grad=True, PyTorch records operations so loss.backward() can compute gradients.

What you should be able to do after this page

Create tensors, move data to device, use autograd, define nn.Module models, train with DataLoader, and write a clean training loop.

3. Hands-on Code

Tensor and autograd

PyTorch tracks operations and computes gradients automatically.

import torch

x = torch.tensor([2.0], requires_grad=True)
y = (x - 3) ** 2
y.backward()
print(x.grad)

Define a model with nn.Module

Keep layers in __init__ and computation in forward.

import torch
from torch import nn

class MLP(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, output_dim),
        )

    def forward(self, x):
        return self.net(x)

model = MLP(10, 32, 2)
print(model)

Training loop skeleton

This loop is the bread and butter. Frameworks change; this shape survives.

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = MLP(10, 32, 2).to(device)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(5):
    model.train()
    total_loss = 0
    for xb, yb in train_loader:
        xb, yb = xb.to(device), yb.to(device)
        optimizer.zero_grad()
        logits = model(xb)
        loss = criterion(logits, yb)
        loss.backward()
        optimizer.step()
        total_loss += loss.item()
    print(epoch, total_loss / len(train_loader))

4. Practical Workflow

A clean PyTorch workflow separates dataset, model, loss, optimizer, training loop, evaluation loop, and checkpointing.

  1. Prepare tensors and DataLoader.
  2. Define model class with explicit input/output dimensions.
  3. Choose loss function matching the task.
  4. Choose optimizer and learning rate.
  5. Train with zero_grad, forward, loss, backward, step.
  6. Evaluate inside torch.no_grad().
  7. Save checkpoints with model state and config.

5. Common Mistakes

Traps to avoid
  • Forgetting optimizer.zero_grad(). Gradients accumulate by default.
  • Using softmax before CrossEntropyLoss. It expects raw logits.
  • Forgetting model.eval() during validation.
  • Mixing CPU and GPU tensors.
  • Using .item() inside too many operations and breaking tensor flow.
