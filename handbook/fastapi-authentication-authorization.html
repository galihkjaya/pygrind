1. Overview

Protect endpoints with API keys, JWT-style users, and role checks. This handbook focuses on practical understanding first: what the tool does, why it exists, how to write the syntax, and how to avoid mistakes that usually waste hours.

Read it like a field guide. You do not need to memorize every command on first pass. You need to understand the shape of the workflow and know which pattern to reach for when a real project asks for it.

2. Mental Model

How to think about this topic

The core mental model: every command or construct has an input, a transformation, and an output. Strong engineers do not just copy syntax. They predict what should happen, run the smallest possible check, then scale the pattern carefully.

Learning target

After this page, you should be able to explain the purpose of the topic, write the main syntax from scratch, debug common failure modes, and apply it inside a small backend or AI-engineering project.

4. Hands-on Code

API key

Good for internal service-to-service calls.

from fastapi import Header, HTTPException, Depends

API_KEY = 'dev-secret'

def require_api_key(x_api_key: str = Header()):
    if x_api_key != API_KEY:
        raise HTTPException(status_code=401, detail='Invalid API key')

@app.get('/internal', dependencies=[Depends(require_api_key)])
def internal():
    return {'ok': True}

Current user dependency

Real JWT code verifies signature, expiry, issuer, and claims.

from fastapi.security import HTTPBearer

bearer = HTTPBearer()

def get_current_user(credentials = Depends(bearer)):
    token = credentials.credentials
    return {'id': 1, 'role': 'admin'}

@app.get('/me')
def me(user: dict = Depends(get_current_user)):
    return user

Role check

Authorization should be explicit near protected routes.

def require_admin(user: dict = Depends(get_current_user)):
    if user['role'] != 'admin':
        raise HTTPException(status_code=403, detail='Admin only')
    return user

@app.post('/admin/courses')
def create_course(user: dict = Depends(require_admin)):
    return {'created_by': user['id']}

5. Practical Workflow

  1. Choose API key for machines or JWT/session for users.
  2. Store secrets in environment variables.
  3. Create a current-user dependency.
  4. Create role/scope dependencies.
  5. Use 401 for not logged in and 403 for not allowed.
  6. Test valid, missing, and invalid credentials.

6. Common Mistakes

Avoid these traps
  • Confusing authn and authz.
  • Storing raw passwords.
  • Putting JWT secrets in frontend code.
  • Trusting JWT payload without verification.
  • Returning 403 when the user is not authenticated at all.
