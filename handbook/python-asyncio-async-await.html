Async Python: asyncio, async & await

Build concurrent I/O programs with coroutines, tasks, timeouts, queues, and async resource management.

1. Mental Model: Concurrency, Not Parallel Magic

Async Python lets one program handle many waiting tasks efficiently. It shines when tasks spend time waiting for I/O: network requests, database calls, files, websockets, API calls. It does not automatically make CPU-heavy code faster. For CPU-heavy work, use multiprocessing, vectorized libraries, or native extensions.

TermMeaning

Event loopScheduler that decides which coroutine runs while others wait

import asyncio

async def main():
    print("Start")
    await asyncio.sleep(1)
    print("Done")

asyncio.run(main())

2. async def and await

async def defines a coroutine function. Calling it does not run it immediately. It returns a coroutine object. The coroutine runs when awaited or scheduled as a task.

import asyncio

async def fetch_user(user_id):
    await asyncio.sleep(0.5)  # pretend network delay
    return {"id": user_id, "name": f"User {user_id}"}

async def main():
    user = await fetch_user(1)
    print(user)

asyncio.run(main())
Common beginner confusion

If you call an async function without await, you do not get the result. You get a coroutine object. Python may warn that it was never awaited.

3. Running Tasks Concurrently

If you await tasks one by one, they run sequentially. To run independent waiting tasks concurrently, use asyncio.gather or create tasks.

import asyncio

async def fetch_page(page):
    await asyncio.sleep(1)
    return f"page-{page}"

async def main():
    results = await asyncio.gather(
        fetch_page(1),
        fetch_page(2),
        fetch_page(3),
    )
    print(results)

asyncio.run(main())

4. create_task: Start Work Now, Await Later

asyncio.create_task() schedules a coroutine to run soon. This is useful when you want work to begin before you await the result.

import asyncio

async def background_save():
    await asyncio.sleep(1)
    print("Saved progress")

async def main():
    task = asyncio.create_task(background_save())
    print("Doing other work...")
    await task

asyncio.run(main())

5. Timeouts and Cancellation

Async systems need timeouts. Without them, one slow request can hang a whole workflow. Cancellation is how asyncio asks a task to stop.

import asyncio

async def slow_api_call():
    await asyncio.sleep(10)
    return "result"

async def main():
    try:
        result = await asyncio.wait_for(slow_api_call(), timeout=1)
        print(result)
    except asyncio.TimeoutError:
        print("Request timed out")

asyncio.run(main())

6. Mixing Sync and Async Code

Blocking sync code inside async code blocks the event loop. If you must run blocking I/O or CPU-light sync code, use asyncio.to_thread(). For serious CPU work, threads may not be enough because of the GIL.

import asyncio
import time

def blocking_work():
    time.sleep(2)
    return "done"

async def main():
    result = await asyncio.to_thread(blocking_work)
    print(result)

asyncio.run(main())

7. Producer-Consumer with asyncio.Queue

asyncio.Queue coordinates async producers and consumers. This pattern appears in crawlers, streaming pipelines, background jobs, and websocket systems.

import asyncio

async def producer(queue):
    for item in range(5):
        await queue.put(item)
    await queue.put(None)  # sentinel

async def consumer(queue):
    while True:
        item = await queue.get()
        if item is None:
            break
        print("Processing", item)
        queue.task_done()

async def main():
    queue = asyncio.Queue()
    await asyncio.gather(producer(queue), consumer(queue))

asyncio.run(main())

8. Async Context Managers

Async resources like HTTP sessions, database pools, or websocket connections often need async setup and cleanup. They use async with.

class AsyncResource:
    async def __aenter__(self):
        print("Opening resource")
        return self

    async def __aexit__(self, exc_type, exc, tb):
        print("Closing resource")
        return False

async def main():
    async with AsyncResource() as resource:
        print("Using resource")

import asyncio
asyncio.run(main())

9. Common Mistakes

  • Using async for CPU-bound code and expecting free speed.
  • Calling async functions without await.
  • Using blocking calls like time.sleep() inside async functions. Use await asyncio.sleep().
  • Creating tasks and never awaiting or tracking them.
  • Skipping timeouts on network calls.
  • Calling asyncio.run() inside an already running event loop.
