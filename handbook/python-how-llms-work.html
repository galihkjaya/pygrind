How LLMs Work: Tokens, Context, Temperature & Prompting

Understand the moving parts before prompting like a wizard with no map.

1. Overview

Large language models predict text token by token using patterns learned from massive training data. Understanding tokens, context windows, sampling, and prompting helps you build more reliable AI features.

Mental model

An LLM receives tokens, builds contextual representations, predicts a distribution over next tokens, samples or selects an output token, appends it, and repeats. It is powerful pattern completion, not a database.

What you should be able to do after this page

Explain tokens, context window, temperature, top-p, system/user messages, prompting patterns, limitations, and why hallucinations happen.

3. Hands-on Code

Prompt structure

Separate role, task, context, constraints, and output format.

system_prompt = "You are a strict Python tutor. Explain concepts clearly."
user_prompt = """
Task: Explain list comprehensions.
Audience: beginner Python learner.
Constraints: include one simple example and one common mistake.
Output: markdown with sections.
"""

Temperature intuition

Lower temperature for deterministic extraction; higher for creative brainstorming.

settings = {
    "extraction": {"temperature": 0.0, "top_p": 1.0},
    "brainstorming": {"temperature": 0.8, "top_p": 0.95},
    "code_review": {"temperature": 0.2, "top_p": 1.0},
}
print(settings)

Few-shot pattern

Examples reduce ambiguity by showing the model the desired mapping.

prompt = """
Classify sentiment as positive, neutral, or negative.

Text: I love this update.
Label: positive

Text: It exists.
Label: neutral

Text: The app keeps crashing.
Label: negative

Text: The UI is cleaner now.
Label:
"""

4. Practical Workflow

Prompting is interface design. You are not begging the model; you are specifying a contract.

  1. Define the task and success criteria.
  2. Provide relevant context but avoid dumping unrelated text.
  3. Specify constraints and output format.
  4. Add examples for ambiguous tasks.
  5. Use lower temperature for factual or structured tasks.
  6. Validate outputs, especially facts and code.
  7. Iterate based on observed failure modes.

5. Common Mistakes

Traps to avoid
  • Treating LLM output as verified truth.
  • Making prompts vague and then blaming the model. Classic.
  • Stuffing too much irrelevant context into the prompt.
  • Using high temperature for extraction or JSON tasks.
  • Forgetting that context windows are finite.
