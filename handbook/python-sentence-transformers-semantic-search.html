Sentence Transformers & Semantic Search

Represent meaning as vectors and search by similarity.

Supports NLP Applications

1. Overview

Sentence Transformers create dense embeddings for sentences, paragraphs, and documents. Semantic search retrieves by meaning instead of exact keyword overlap, making it useful for FAQ search, recommendations, clustering, and RAG.

Mental model

Embedding models map text into vector space. Similar meanings should land near each other. Search becomes nearest-neighbor lookup using cosine similarity or vector indexes.

What you should be able to do after this page

Generate sentence embeddings, compute cosine similarity, build a small semantic search engine, understand chunking, normalization, and retrieval evaluation.

3. Hands-on Code

Generate embeddings

One vector per sentence or document chunk.

from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")
docs = [
    "Python lists store ordered items.",
    "Dictionaries map keys to values.",
    "Neural networks learn from gradients.",
]
embeddings = model.encode(docs, normalize_embeddings=True)
print(embeddings.shape)

Brute-force semantic search

For small collections, matrix multiplication is enough.

import numpy as np

query = "How do key value mappings work?"
q_emb = model.encode([query], normalize_embeddings=True)
scores = q_emb @ embeddings.T
best = np.argsort(scores[0])[::-1]

for idx in best:
    print(float(scores[0, idx]), docs[idx])

Chunk long documents

Embedding whole giant documents usually loses detail.

def simple_chunks(text, max_words=120):
    words = text.split()
    return [" ".join(words[i:i+max_words]) for i in range(0, len(words), max_words)]

chunks = simple_chunks(long_document)
chunk_embeddings = model.encode(chunks, normalize_embeddings=True)

4. Practical Workflow

Semantic search quality depends on chunking, embedding model choice, metadata, and evaluation queries.

  1. Collect documents and clean obvious noise.
  2. Split long documents into meaningful chunks.
  3. Embed chunks and store vectors plus metadata.
  4. Embed user query.
  5. Retrieve top-k by similarity.
  6. Rerank or filter if needed.
  7. Evaluate with test queries and expected relevant docs.

5. Common Mistakes

Traps to avoid
  • Embedding documents that are too long and losing specific facts.
  • Ignoring metadata such as source, date, and section.
  • Assuming high similarity always means correct answer.
  • Using English-only embeddings for multilingual content.
  • Not evaluating retrieval with realistic queries.
